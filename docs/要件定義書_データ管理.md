## 1. 概要

### 1.1 目的
小規模多機能型居宅介護事業所の利用調整を行うため、以下のデータを管理する：
- 利用者マスタ
- 月間スケジュール（33セル構造）
- 定員チェック結果
- 備考・メモ

### 1.2 データ保存方式
- **Phase 0〜1**: LocalStorage（ブラウザローカル）
- **Phase 2以降**: サーバー保存を検討

### 1.3 データ保持期間
- **保持期間**: 1ヶ月分のみ
- **自動削除**: 古い月のデータを自動削除（容量管理）

## 2. データ構造

### 2.1 利用者マスタ

#### データ項目

| 項目名 | 型 | 必須 | 説明 |
|--------|-----|------|------|
| id | string | ○ | 利用者ID（`user001`形式） |
| name | string | ○ | 氏名 |

#### 保存キー
```
schedule_users
```

#### データ例
```json

  "id": "user001",
  "name": "青砥美秋",
  "registrationDate": "2025-01-15",
  "sortId": 0,
  "note": "毎月第3週は家族旅行",
  "isActive": true
}
```

### 2.2 月間スケジュール

#### データ項目

| 項目名 | 型 | 必須 | 説明 |
|--------|-----|------|------|
| userId | string | ○ | 利用者ID |
| yearMonth | string | ○ | 年月（`YYYY-MM`） |
| cells | Map | ○ | セルデータ（33セル × 2行） |
| stayPeriods | Array | ○ | 宿泊期間リスト |

#### セル構造（33セル）

**通泊統合行（dayStay）**:
```
[特別セル] [1日] [2日] ... [31日] [特別セル]
   ↑                              ↑
  前月                            翌月
```

**訪問行（visit）**:
```
[特別セル] [1日] [2日] ... [31日] [特別セル]
```

#### セルデータ項目

| 項目名 | 型 | 必須 | 説明 |
|--------|-----|------|------|
| userId | string | ○ | 利用者ID |
| date | string | ○ | 日付（`YYYY-MM-DD`または特別セル） |
| cellType | string | ○ | `"dayStay"` または `"visit"` |
| inputValue | string | - | 入力値（記号または数値） |
| note | string | - | セル備考 |
| actualFlags | object | ○ | 実績フラグ |
| deletedValue | string | - | 削除前の値（復元用） |
| deletedAt | number | - | 削除日時（UnixTime） |

#### 記号一覧（dayStayセル）

| 記号 | 意味 | Unicode |
|------|------|---------|
| ○ | 終日（通いまたは宿泊中） | U+25CB |
| ◓ | 前半のみ | U+25D3 |
| ◒ | 後半のみ | U+25D2 |
| 入 | 入所 | - |
| 退 | 退所 | - |

#### 特別セルの形式

**前月セル**:
```
date: "2025-12-prevMonth"
inputValue: "11/28入"  // 前月28日に入所
```

**翌月セル**:
```
date: "2025-12-nextMonth"
inputValue: "1/3退"    // 翌月3日に退所
```

#### 保存キー
```
schedule_{yearMonth}
例: schedule_2025-12
```

#### データ例

## 3. CSV取り込み仕様

### 3.1 CSVファイル形式

#### ファイル情報
- **エンコーディング**: Shift_JIS
- **区切り文字**: カンマ（`,`）
- **引用符**: ダブルクォート（`"`）
- **改行**: CRLF（`\r\n`）
- **出力元**: 介護ソフト（CAREKARTE）の「算定一覧」

#### 列構造

| 列番号 | 列名 | 説明 | 例 |
|--------|------|------|-----|
| 0 | 対象フラグ | 常に`False` | `False` |
| 1 | 利用者名 | 氏名（先頭行のみ） | `青砥美秋` |
| 2 | 時間 | サービス提供時間 | `11:00` |
| 3 | 基本 | 常に`基本` | `基本` |
| 4 | サービスコード | 6桁の数値 | `071111` |
| 5 | 月曜 | 週間パターン | `1` または `-` |
| 6 | 火曜 | 週間パターン | `1` または `-` |
| 7 | 水曜 | 週間パターン | `1` または `-` |
| 8 | 木曜 | 週間パターン | `1` または `-` |
| 9 | 金曜 | 週間パターン | `1` または `-` |
| 10 | 土曜 | 週間パターン | `1` または `-` |
| 11 | 日曜 | 週間パターン | `1` または `-` |
| 12-15 | その他 | （未使用） | - |

#### CSV例
```csv
False,青砥美秋,"11:00",基本,071111,1,-,-,-,1,1,1,,,,
,,"11:00",基本,071111,-,1,1,1,-,-,-,,,,
,,"11:00",基本,061111,-,-,1,-,-,1,-,,,,
,,"07:00",基本,081111,1,1,1,1,1,1,1,,,,
False,安藤敏子,"11:00",基本,061111,1,-,-,-,-,1,-,,,,
```

**構造の特徴**:
1. **利用者名は先頭行のみ** - 2行目以降は空セル
2. **同一利用者の複数行** - 1人につき1〜20行程度
3. **サービス種別ごとに行が分かれる** - 通所・訪問・宿泊が別々の行

---

### 3.2 サービスコード分類

#### 判定ルール
**上2桁（先頭2文字）で判定**:

| コード範囲 | サービス種別 | 展開先 | 例 |
|-----------|------------|--------|-----|
| `06xxxx` | 訪問介護等 | visitPattern | `061111`, `061121` |
| `07xxxx` | 通所介護等（通い） | dayPattern | `071111`, `071121` |
| `08xxxx` | 短期入所等（宿泊） | stayPattern | `081111`, `081131` |

**下4桁は事業所コード等として扱い、判定には使用しない**

#### 例
```
071111: 通所介護（事業所1） → dayPattern
071121: 通所介護（事業所2） → dayPattern
061111: 訪問介護（事業所1） → visitPattern
081111: 短期入所（事業所1） → stayPattern
```

---

### 3.3 週間パターン抽出

#### 処理フロー

**Step 1: 利用者ごとにグループ化**
```
利用者名がある行 → 新しい利用者の開始
利用者名が空の行 → 直前の利用者の続き
```

**Step 2: サービス種別ごとに週間パターンを統合**
```javascript
// 同じ利用者の複数行を統合
{
  name: "青砥美秋",
  dayPattern: [月, 火, 水, 木, 金, 土, 日],    // 07で始まる行を統合
  visitPattern: [月, 火, 水, 木, 金, 土, 日],  // 06で始まる行を統合
  stayPattern: [月, 火, 水, 木, 金, 土, 日]    // 08で始まる行を統合
}
```

#### 統合ルール

**通所・宿泊（OR結合）**:
```
行1: [1, -, -, -, 1, 1, 1]
行2: [-, 1, 1, 1, -, -, -]
結果: [1, 1, 1, 1, 1, 1, 1]  ← どちらかが1なら1
```

**訪問（回数加算）**:
```
行1: [1, -, -, -, 1, 1, 1]
行2: [-, 1, 1, 1, -, -, -]
結果: [1, 1, 1, 1, 1, 1, 1]  ← 回数を加算
```

### 3.4 月間展開

#### 処理フロー

**Step 1: 週間パターンを月間に展開**
```javascript
// 2025年11月の場合
for (let day = 1; day <= 30; day++) {
  const date = `2025-11-${String(day).padStart(2, '0')}`;
  const dayOfWeek = new Date(date).getDay(); // 0=日曜, 6=土曜
  const patternIndex = dayOfWeek === 0 ? 6 : dayOfWeek - 1; // 月=0, ..., 日=6
  
  // 通所
  if (dayPattern[patternIndex] === '1') {
    calendar.setCell(date, 'dayStay', '○');
  }
  
  // 訪問
  if (visitPattern[patternIndex] !== '-') {
    calendar.setCell(date, 'visit', visitPattern[patternIndex]);
  }
  
  // 宿泊（連続期間として処理）
  if (stayPattern[patternIndex] === '1') {
    // 入所・退所の判定は後で行う
  }
}
```

**Step 2: 宿泊期間を調整**
```javascript
// 連続する宿泊を「入→○→退」に変換
let inStay = false;
for (const date of dates) {
  const cell = calendar.getCell(date, 'dayStay');
  if (cell && cell.inputValue === '○' && stayFlag) {
    if (!inStay) {
      cell.inputValue = '入';  // 初日
      inStay = true;
    } else {
      // 2日目以降は'○'のまま
    }
  } else if (inStay) {
    // 前日まで宿泊だった場合、前日を'退'に
    const prevDate = DateUtils.addDays(date, -1);
    const prevCell = calendar.getCell(prevDate, 'dayStay');
    if (prevCell) {
      prevCell.inputValue = '退';
    }
    inStay = false;
  }
}
```

### 3.5 備考の保持

CSV取り込み時、既存のセル備考は保持されます。

#### 処理フロー
```javascript
// 既存のスケジュールがある場合
const existingSchedule = await StorageService.loadSchedule(yearMonth);

if (existingSchedule) {
  // 新しいカレンダーに備考をマージ
  for (const [userId, calendar] of calendars) {
    const existingCalendar = existingSchedule.get(userId);
    if (existingCalendar) {
      for (const [key, cell] of calendar.cells) {
        const existingCell = existingCalendar.cells.get(key);
        if (existingCell && existingCell.note) {
          cell.note = existingCell.note;  // 備考を引き継ぐ
        }
      }
    }
  }
}
```

## 4. データ保存・読込

### 4.1 保存タイミング

| 操作 | 保存対象 | タイミング |
|------|----------|-----------|
| セル入力 | スケジュール | 入力確定時（即時） |
| 備考入力 | 備考データ | 入力確定時（即時） |
| CSV取り込み | 利用者マスタ、スケジュール | 取り込み完了時 |
| 利用者追加 | 利用者マスタ | 追加時（即時） |

### 4.2 自動保存

**Phase 0の仕様**:
- セル入力ごとに自動保存
- LocalStorageに即座に書き込み
- 保存エラー時はユーザーに通知

## 5. データ容量管理

### 5.1 保持期間

**1ヶ月分のみ保持**:
```javascript
// 古い月を自動削除
const currentMonth = DateUtils.getCurrentYearMonth();
const keys = Object.keys(localStorage);

keys.forEach(key => {
  if (key.startsWith('schedule_') && key !== `schedule_${currentMonth}`) {
    localStorage.removeItem(key);
    Logger.info(`Deleted old schedule: ${key}`);
  }
});
```

### 5.2 容量超過時の対応

**Phase 0の対応**:
1. 古い月のデータを削除
2. ユーザーに警告表示
3. エクスポート機能の案内（Phase 2で実装）

## 6. エラーハンドリング

### 6.1 CSV取り込みエラー

| エラー種別 | 検出条件 | 対応 |
|-----------|---------|------|
| ファイル形式エラー | CSVではない | ダイアログで通知 |
| 文字コードエラー | Shift_JISで読めない | エラー表示 |
| 列数不足 | 16列未満 | 行番号を表示 |
| サービスコード不正 | 6桁の数値でない | スキップして警告 |

### 6.2 保存エラー

| エラー種別 | 検出条件 | 対応 |
|-----------|---------|------|
| LocalStorage満杯 | QuotaExceededError | 古いデータ削除を提案 |
| 書き込み失敗 | 例外発生 | エラーダイアログ表示 |
