## 📋 ドキュメント作成ガイド

### テンプレートとチェックリスト

プロジェクトBでは、前プロジェクトの教訓を活かし、設計品質を担保するためのテンプレートとチェックリストを用意しています。

**配置場所**: `/mnt/project/templates/`

| ファイル名 | 用途 |
|-----------|------|
| **TEMPLATE_L1_概要.md** | L1層の概要ドキュメント作成時 |
| **TEMPLATE_L2_データ構造.md** | L2層のデータ構造ドキュメント作成時 |
| **TEMPLATE_L2_UI設計.md** | L2層のUI設計ドキュメント作成時 |
| **TEMPLATE_L2_ロジック.md** | L2層のロジックドキュメント作成時 |
| **CHECKLIST_設計レビュー.md** | 各ドキュメントのレビュー時 |

### ドキュメント作成の手順

1. **テンプレートを参照**
   - 該当する層のテンプレートをコピー
   
2. **内容を執筆**
   - テンプレートのセクションに沿って記述
   - 前プロジェクトの教訓を意識
   
3. **設計チェックリストでレビュー**
   - CHECKLIST_設計レビュー.md を使用
   - すべての項目をチェック
   
4. **ユーザーと確認**
   - 曖昧な部分がないか対話で確認
   
5. **完成・次のドキュメントへ**

## 🎯 プロジェクトBの設計原則（再掲）

1. **月別予定表の実務が唯一の基準** - CSV/Excelは入出力手段
2. **各セクションは独立** - 通い・泊まり・訪問は独立設計
3. **日付軸思考** - 「この日は誰？」に答える設計
4. **居室管理重視** - 泊まりセクションは居室が主役
5. **拡張性** - セクション追加・統合を前提
6. **CAREKARTEとの補完関係** - 予定調整と記録・請求の役割分担
