## 1. 利用者マスタ

### 1.1 起動時の状態

**確定事項**（2025-11-14）:
- ✅ 利用者マスタは**空の状態**で起動
- ✅ DEFAULT_USERSは**廃止**

**理由**:
- 実際の運用では施設ごとに利用者が異なる
- CSV取り込みで実データを読み込む前提

---

### 1.2 利用者の追加

#### 1.2.1 CSV取り込み時

**処理フロー**:
```
1. CSVから利用者名を抽出
2. 重複を除外（同じ利用者の複数行をまとめる）
3. あいうえお順にソート
4. 内部IDを自動採番（user001, user002, ...）
5. sortIdを付与（0, 1, 2, ...）
6. 利用者マスタに登録
```

**既存利用者がいる場合**:
- Phase 2で実装予定の「利用者確認画面」を表示
- Phase 1では上書き（警告のみ）

---

#### 1.2.2 内部ID（userId）

**形式**: `user001`, `user002`, ..., `user029`

**採番ルール**:
- 3桁ゼロパディング
- あいうえお順に採番
- 削除されたIDは再利用しない（Phase 2で検討）

**実装例**:
```javascript
const userId = `user${String(index + 1).padStart(3, '0')}`;
```

---

#### 1.2.3 sortId

**用途**: 表示順序の制御

**初期値**: CSV取り込み時にあいうえお順で0, 1, 2, ...

**将来拡張**（Phase 2以降）:
- ドラッグ&ドロップで並び替え
- sortIdを更新して順序を保存

---

### 1.3 利用者の削除

**Phase 1**: 未実装

**Phase 2**（検討中）:
- 論理削除（isActive = false）
- データは残すが画面に表示しない
- 過去の予定データとの整合性を保つ

## 2. 月間スケジュール

### 2.1 データ保持期間

**確定事項**（2025-11-14）:
- ✅ **1ヶ月分のみ**保持
- ✅ 古い月は**自動削除**

**理由**:
- LocalStorageの容量制限（Safari: 2.5MB）
- 1ヶ月分 ≒ 600KB（29名 × 33セル × 2行）
- 4ヶ月分まで保持可能だが、余裕を持って1ヶ月のみ

---

### 2.2 月の切り替え

**動作**:
```
1. 月選択ドロップダウンで月を選択
2. LocalStorageから該当月のデータを読み込み
   ├─ データあり → 表示
   └─ データなし → 空のグリッド表示
3. 前の月のデータは保持（削除しない）
```

**自動削除タイミング**:
- 2ヶ月前のデータを削除
- 例：2025年12月を表示中 → 2025年10月以前を削除

---

### 2.3 データ保存タイミング

**確定事項**（2025-11-14）:
- ✅ **セル入力ごとに自動保存**

**理由**:
- ブラウザクラッシュ時のデータ損失を防ぐ
- ユーザーが保存を忘れるリスクを回避

**実装**:
```javascript
// セル編集時
cell.inputValue = newValue;
calendar.save();  // 自動保存
```

#### 3.2.4 サービスコード

**基本構造**: 6桁の数字（例：071111）

| コード例 | サービス種別 | 意味 |
|---------|-------------|------|
| 061111 | 訪問介護等 | 訪問サービス |
| 071111 | 通所介護等 | 通いサービス |
| 081111 | 短期入所等 | 宿泊サービス |
| 731xxx | 小規模多機能（基本） | 月額包括報酬 |
| 734xxx, 736xxx | 加算 | 各種加算 |

---

#### 3.2.4-1 サービスコードの判定ルール ★v1.1で追加

**介護保険のサービスコードは上2桁でサービス種別を分類します**:

| 上2桁 | サービス種別 | 本システムでの扱い |
|-------|-------------|------------------|
| **06** | 訪問介護等 | 訪問（visitPattern） |
| **07** | 通所介護等 | 通い（dayPattern） |
| **08** | 短期入所等 | 宿泊（stayPattern） |
| 73, 74, 76 | 加算・基本報酬 | 無視（週間パターンには含めない） |

**判定方法**:
```javascript
const prefix = serviceCode.substring(0, 2);

if (prefix === '06') {
  // 訪問介護等 → visitPatternに設定
  pattern.visitPattern = weekPattern.map(v => v || '0');
} else if (prefix === '07') {
  // 通所介護等 → dayPatternに設定（通い）
  pattern.dayPattern = weekPattern.map(v => v === '1' ? '1' : '-');
} else if (prefix === '08') {
  // 短期入所等 → stayPatternに設定（宿泊）
  pattern.stayPattern = weekPattern.map(v => v === '1' ? '1' : '-');
}
```

**重要な注意点**:
- ✅ **上2桁で判定**する（下4桁は事業所コード等のため使用しない）
- ❌ 末尾の111/121/131では判定しない（誤り）
- ✅ 加算コード（73xxxx等）は週間パターンに含めない

**実装例（誤り）**:
```javascript
// ❌ 間違い：末尾3桁で判定
if (serviceCode.endsWith('111')) {
  // 通所として処理 → 061111も誤って通所になる
}
```

**実装例（正しい）**:
```javascript
// ✅ 正しい：上2桁で判定
const prefix = serviceCode.substring(0, 2);
if (prefix === '07') {
  // 通所として処理 → 071111のみ該当
}
```

---

#### 3.2.5 曜日の値

| 値 | 意味 |
|----|------|
| `-` | 利用なし |
| `1` | 利用あり（通い・宿泊） |
| `2`, `3`, ... | 訪問回数 |
| 空白 | 利用なし |

### 3.4 週間パターン→月間展開

#### 3.4.1 アルゴリズム（概要）

```
入力: 
  - userPatterns: 週間パターン（月〜日）
  - yearMonth: 対象月（例：'2025-12'）

処理:
  1. 対象月の全日付を取得（1〜31日）
  
  2. 各日付の曜日を判定（月〜日）
  
  3. 週間パターンから対応する値を取得
     - 通い: dayPattern[曜日]
     - 訪問: visitPattern[曜日]
     - 宿泊: stayPattern[曜日]
  
  4. セル値を設定
     - 通い: "○"
     - 訪問: 回数（"1", "2", ...）
     - 宿泊: 仮で"入"（後で調整）
  
  5. 連続する宿泊を「入所〜退所」に調整
     - 連続する日 → 開始日「入」、中間日「○」、終了日「退」
     - 単独の日 → 「入」と「退」を同日に

出力:
  - calendar: ScheduleCalendar形式の月間予定
```

**詳細な実装はExcelServiceのexpandWeeklyToMonthly()メソッドに委譲**

---

#### 3.4.2 宿泊期間の調整ロジック（概要）

```
入力: 
  - userCalendar: 利用者の月間カレンダー（仮の"入"が設定済み）

処理:
  1. 通泊行のセルを日付順にソート
  
  2. 連続する宿泊日をグループ化
     - 日付の間隔が1日 → 連続
     - 日付の間隔が2日以上 → 別グループ
  
  3. 各グループに対して
     - 開始日 → "入"
     - 中間日 → "○"（通い+宿泊）
     - 終了日 → "退"
  
  4. 単独の宿泊日（前後に宿泊なし）
     - 開始日 → "入"
     - 終了日（同日） → "退"

出力:
  - 調整済みのuserCalendar
```

**詳細な実装はScheduleCalendarのcalculateStayPeriods()メソッドに委譲**

### 3.5 上書き時の動作

**確定事項**（2025-11-14）:
- ✅ **備考は保持**
- ✅ 予定データは上書き

**処理フロー**（Phase 2で実装）:
```javascript
async function handleCSVImport(files) {
  const hasExistingData = checkExistingSchedule();
  
  if (hasExistingData) {
    const confirmed = await showConfirmDialog(
      '既存の予定データは上書きされます。\n' +
      '備考は保持されます。\n\n' +
      '算定基礎を取り込みますか？'
    );

  }
  
  // 読み込み処理
  const userPatterns = await parseCSVFiles(files);
  const calendar = expandWeeklyToMonthly(userPatterns, currentYearMonth);
  
  // 備考を保持したまま予定を上書き
  const mergedCalendar = mergeWithExistingNotes(calendar);
  
  saveSchedule(currentYearMonth, mergedCalendar);
  renderGrid();
  
  showToast('算定基礎を取り込みました', 'success');
}
```

### 4.3 容量管理

## 5. 実装時の注意事項

### 1. CSV取り込み
- Shift_JISの正しい読み込み（文字化け対策）
- **引用符内改行の処理**（RFC 4180準拠）
- 複数CSVファイル（介護+予防）の統合
- **サービスコードは上2桁で判定**（v1.1で追加）
- 同姓同名の慎重な扱い

- 内部IDの採番ルールの厳守
- sortIdの一貫性

- LocalStorage容量超過時の自動削除
- 月の切り替え時のデータクリア
- 備考の保持（上書き時）
- データの整合性チェック
- **すべてのキーにプレフィックスを使用**（v1.1で追加）

### 4. エラーハンドリング
- CSV読み込みエラー
- LocalStorage書き込みエラー
