# 命名規則ガイド

## 1. 概要

このドキュメントは、小規模多機能利用調整システムにおける命名規則を定義します。
一貫性のある命名により、コードの可読性・保守性を向上させます。

---

## 2. クラス名

### 2.1 基本ルール

```javascript
// ✅ 良い例: PascalCase、名詞
class ScheduleController { }
class StorageService { }
class ScheduleGrid { }

// ❌ 悪い例
class scheduleController { }  // camelCaseは使わない
class ControlSchedule { }     // 動詞始まりは使わない
class Schedule { }            // 曖昧（何のSchedule?）
```

### 2.2 サフィックスによる区別

| サフィックス | 用途 | 例 |
|-------------|------|-----|
| Controller | ビジネスロジック管理 | ScheduleController |
| Service | インフラ層（永続化、外部連携）| StorageService |
| （なし） | UIコンポーネント | ScheduleGrid |
| （なし） | ドメインモデル | User, Note |

## 3. メソッド名

### 3.1 基本ルール

```javascript
// ✅ 良い例: camelCase、動詞始まり
getCurrentYearMonth()
loadSchedule()
saveUsers()

// ❌ 悪い例
GetCurrentYearMonth()  // PascalCaseは使わない
current_year_month()   // snake_caseは使わない
yearMonth()            // 動詞がない
```

### 3.2 動詞の使い分け

| 動詞 | 意味 | 例 |
|------|------|-----|
| get | 取得（計算なし、既存値を返す） | getUser(id) |
| calculate | 計算して取得 | calculateFTE() |
| find | 検索して取得 | findStaffByQualification() |
| load | 外部から読み込み | loadSchedule() |
| save | 外部に保存 | saveSchedule() |
| create | 新規作成 | createNote() |
| update | 更新 | updateCell() |
| delete | 削除 | deleteNote() |
| is / has | 真偽値を返す | isFullTime(), hasNote() |

### 3.3 命名パターン

#### 現在の値を取得
```javascript
// ✅ 正しい
getCurrentYearMonth()  // "2025-12"を返す
getCurrentDate()       // Date objectを返す

// ❌ 間違い
getCurrentMonth()      // JavaScriptのgetMonth()と混同
getToday()            // 一般的でない
```

#### 全体を取得
```javascript
// ✅ 正しい
getAllUsers()          // 全ユーザー
getAllDates()          // 全日付

// ❌ 間違い
getUsers()             // 単数形は1つ、複数形は全体
```

#### 条件付き取得
```javascript
// ✅ 正しい
getActiveUsers()       // 有効なユーザーのみ
getOverCapacityDates() // 定員オーバーの日付のみ

// ❌ 間違い
getUsersActive()       // 形容詞は前に
```

#### 真偽値を返す
```javascript
// ✅ 正しい
isEmpty()
isFullTime()
hasNote()
isOverCapacity()

// ❌ 間違い
empty()        // 動詞がない
checkEmpty()   // isの方が簡潔
```

## 4. 変数名

### 4.1 基本ルール

```javascript
// ✅ 良い例: camelCase、名詞
const userName = "山田太郎";
const scheduleController = new ScheduleController();
const isActive = true;

// ❌ 悪い例
const UserName = "山田太郎";          // PascalCaseは使わない
const schedule_controller = ...;      // snake_caseは使わない
const active = true;                  // 真偽値はis/hasを付ける
```

### 4.2 真偽値

```javascript
// ✅ 正しい
const isActive = true;
const hasNote = false;
const canEdit = true;

// ❌ 間違い
const active = true;     // is/hasがない
const noteExists = true; // existsよりhasが簡潔
```

### 4.3 配列とMap

```javascript
// ✅ 正しい
const users = [];              // 複数形
const calendars = new Map();   // 複数形
const dates = [];              // 複数形

// ❌ 間違い
const userList = [];           // Listサフィックスは冗長
const userArray = [];          // Arrayサフィックスは冗長
```

### 4.4 定数

```javascript
// ✅ 正しい: UPPER_SNAKE_CASE
const MAX_CAPACITY = 15;
const DEFAULT_YEAR_MONTH = "2025-12";

// ❌ 間違い
const maxCapacity = 15;        // 定数はUPPER_SNAKE_CASE
```

## 5. ファイル名

### 5.1 基本ルール

```
✅ 正しい: PascalCase（クラス名と一致）
ScheduleController.js
StorageService.js
ScheduleGrid.js

❌ 間違い
scheduleController.js   // camelCaseは使わない
schedule-controller.js  // kebab-caseは使わない
```

### 5.2 特殊なファイル

```
小文字: 設定ファイル、エントリーポイント
config.js
main.js
index.html

複数形: データファイル
users.js
shiftPatterns.js
```

## 6. イベント名

### 6.1 基本ルール

```javascript
// ✅ 正しい: camelCase、過去形
this.emit('cellUpdated', data);
this.emit('scheduleLoaded', data);
this.emit('monthChanged', data);

// ❌ 間違い
this.emit('cell-updated', data);  // kebab-caseは使わない
this.emit('cellUpdate', data);    // 現在形ではなく過去形
this.emit('CELL_UPDATED', data);  // UPPER_SNAKEは使わない
```

### 6.2 命名パターン

| パターン | 説明 | 例 |
|---------|------|-----|
| xxxUpdated | データが更新された | cellUpdated, scheduleUpdated |
| xxxLoaded | データが読み込まれた | scheduleLoaded, usersLoaded |
| xxxChanged | 状態が変更された | monthChanged, userChanged |
| xxxDeleted | データが削除された | noteDeleted, userDeleted |
| xxxAdded | データが追加された | noteAdded, userAdded |

## 7. CSS クラス名

### 7.1 基本ルール

```css
/* ✅ 正しい: kebab-case */
.schedule-grid { }
.cell-editor { }
.user-name { }

/* ❌ 間違い */
.scheduleGrid { }    /* camelCaseは使わない */
.schedule_grid { }   /* snake_caseは使わない */
.ScheduleGrid { }    /* PascalCaseは使わない */
```

### 7.2 BEM風の命名

```css
/* Block */
.schedule-grid { }

/* Element */
.schedule-grid-cell { }
.schedule-grid-header { }

/* Modifier */
.schedule-grid-cell.editing { }
.schedule-grid-cell.error { }
```

## 10. よくある間違いと修正例

### 10.1 JavaScript標準メソッドとの混同

```javascript
// ❌ 間違い: JavaScriptのgetMonth()は0-11を返す
getCurrentMonth() {
  return new Date().getMonth();  // 0-11が返る
}

// ✅ 正しい: 明確な命名で混乱を避ける
getCurrentYearMonth() {
  const now = new Date();
  return `${now.getFullYear()}-${String(now.getMonth() + 1).padStart(2, '0')}`;
}
```

### 10.2 曖昧な命名

```javascript
// ❌ 曖昧
const data = getData();        // 何のデータ?
const result = check();        // 何をチェック?

// ✅ 明確
const scheduleData = getScheduleData();
const capacityCheckResult = checkCapacity();
```

### 10.3 冗長な命名

```javascript
// ❌ 冗長
const userList = [];
const userArray = [];
getUserArray()

// ✅ 簡潔
const users = [];
getUsers()
```

## 11. チェックリスト

コードレビュー時に以下を確認：

- [ ] クラス名はPascalCase、名詞
- [ ] メソッド名はcamelCase、動詞始まり
- [ ] 変数名はcamelCase、名詞
- [ ] 真偽値はis/hasで始まる
- [ ] 定数はUPPER_SNAKE_CASE
- [ ] ファイル名はクラス名と一致
- [ ] イベント名は過去形
- [ ] CSSクラス名はkebab-case
- [ ] HTML IDはkebab-case
- [ ] 曖昧・冗長な命名はない
