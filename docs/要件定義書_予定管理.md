## 1. 通泊行の記号定義

### 1.1 基本記号

| 記号 | Unicode | 意味 | 定員カウント |
|------|---------|------|-------------|
| **○** | U+25CB | 通い全日 | 前半1 + 後半1 |
| **◓** | U+25D3 | 前半通い | 前半1 + 後半0 |
| **◒** | U+25D2 | 後半通い | 前半0 + 後半1 |
| **入** | - | 入所（宿泊開始） | 前半1 + 後半1 |
| **退** | - | 退所（宿泊終了） | 前半1 + 後半1（泊りは+0） |
| 空白（青背景） | - | 宿泊中 | 前半1 + 後半1 |

---

### 1.2 記号の詳細説明

#### 1.2.1 通い全日（○）

**意味**:
- 通いサービスを全日利用
- 前半（午前）と後半（午後）の両方を利用

**定員カウント**:
- 通い定員：前半+1、後半+1
- 泊り定員：+0

**使用頻度**:
- 最も一般的
- 基本的な通いサービス

**入力方法**:
- クリック（空白 ↔ ○ トグル）
- 右クリックメニュー

---

#### 1.2.2 前半通い（◓）

**意味**:
- 通いサービスを前半（午前）のみ利用
- 後半（午後）は利用しない

**定員カウント**:
- 通い定員：前半+1、後半+0
- 泊り定員：+0

**使用頻度**:
- レアケース
- 必要になれば、その状況が解消するまでは定期的に発生

**時刻定義**:
- 明確な時刻は定義しない
- 「時間が被らないように利用してください」という意図

**入力方法**:
- **右クリックメニューのみ**
- クリックでは入力不可（誤爆防止）

---

#### 1.2.3 後半通い（◒）

**意味**:
- 通いサービスを後半（午後）のみ利用
- 前半（午前）は利用しない

**定員カウント**:
- 通い定員：前半+0、後半+1
- 泊り定員：+0

**使用頻度**:
- レアケース
- 必要になれば、その状況が解消するまでは定期的に発生

**入力方法**:
- **右クリックメニューのみ**
- クリックでは入力不可（誤爆防止）

---

#### 1.2.4 入所（入）

**意味**:
- 宿泊サービスの開始日
- この日から宿泊期間が始まる

**定員カウント**:
- 通い定員：前半+1、後半+1
- 泊り定員：+1

**自動設定**:
- ドラッグ操作で自動設定される
- 手動入力も可能（右クリックメニュー）

**視覚表現**:
- 文字：「入」
- 背景色：薄青（#d0e8f0）
- 左罫線：青（#0066cc）、太さ3px

---

#### 1.2.5 退所（退）

**意味**:
- 宿泊サービスの終了日
- この日で宿泊期間が終わる

**定員カウント**:
- 通い定員：前半+1、後半+1
- **泊り定員：+0**（★重要：退所日は泊りカウントしない）

**理由**:
- 退所日は朝に退所するため、その日の夜は宿泊しない
- 通いサービスとしてのみカウント

**自動設定**:
- ドラッグ操作で自動設定される
- 手動入力も可能（右クリックメニュー）

**視覚表現**:
- 文字：「退」
- 背景色：薄青（#d0e8f0）
- 左罫線：青（#0066cc）、太さ3px

---

#### 1.2.6 宿泊中（○、青背景）

**意味**:
- 入所〜退所の間の日
- 通いと宿泊の両方を利用

**定員カウント**:
- 通い定員：前半+1、後半+1
- 泊り定員：+1

**自動設定**:
- 入所〜退所を設定すると自動的に「○」が設定される
- calculateStayPeriods()で計算

**視覚表現**:
- 文字：「○」
- 背景色：極薄青（#e8f4f8）
- 左罫線：青（#0066cc）、太さ3px

**例**:
```
┌────┬───┬───┬───┬───┐
│青柳│入  │ ○ │ ○ │退  │
│美秋│├青│├青│├青│├青│
└────┴───┴───┴───┴───┘
       ↑   ↑   ↑   ↑
     入所 宿泊 宿泊 退所
```

---

### 1.3 記号の入力方法まとめ

| 記号 | クリック | 右クリックメニュー | ドラッグ | 自動設定 |
|------|---------|-------------------|---------|---------|
| ○ | ✅ | ✅ | ❌ | ❌ |
| ◓ | ❌ | ✅ | ❌ | ❌ |
| ◒ | ❌ | ✅ | ❌ | ❌ |
| 入 | ❌ | ✅ | ✅ | ❌ |
| 退 | ❌ | ✅ | ✅ | ❌ |
| ○（宿泊中） | ❌ | ❌ | ✅ | ✅ |

## 2. 訪問回数の入力

### 2.1 基本仕様

**入力方法**:
- クリック連打：0→1→2→...→9→0
- 右クリックメニュー：直接入力（10回以上も可能）

**上限**:
- 理論値：制限なし（10回、15回など対応）
- 実運用：9回以上は事実上ない

---

### 2.2 クリック連打

#### 2.2.1 動作

**状態遷移**:
```
空白 → 1 → 2 → 3 → 4 → 5 → 6 → 7 → 8 → 9 → 空白
```

**実装**:
```javascript
handleVisitCellClick(cell) {
  const currentValue = parseInt(cell.inputValue) || 0;
  const nextValue = (currentValue + 1) % 10;
  
  if (nextValue === 0) {
    cell.inputValue = '';
  } else {
    cell.inputValue = String(nextValue);
  }
  
  this.saveCell(cell);
  this.renderCell(cell);
}
```

---

#### 2.2.2 使用例

**シナリオ1: 訪問2回**
```
1回目クリック: 空白 → 1
2回目クリック: 1 → 2
```

**シナリオ2: 訪問回数の修正（3回→1回）**
```
1回目クリック: 3 → 4
2回目クリック: 4 → 5
3回目クリック: 5 → 6
4回目クリック: 6 → 7
5回目クリック: 7 → 8
6回目クリック: 8 → 9
7回目クリック: 9 → 空白
8回目クリック: 空白 → 1
```

**改善案**（将来的に検討）:
- Shift+クリック：-1（逆方向）
- Ctrl+クリック：クリア（空白に戻す）

---

### 2.3 右クリックメニュー

#### 2.3.1 メニュー内容

```
┌──────────┐
│ 1回       │ → 1
│ 2回       │ → 2
│ 3回       │ → 3
│ 4回       │ → 4
│ 5回       │ → 5
│ 直接入力  │ → テキスト入力ダイアログ
│ クリア    │ → 空白
│ --------- │
│ 備考      │ → 備考パネル表示
└──────────┘
```

---

#### 2.3.2 直接入力ダイアログ

**表示内容**:
```
┌─────────────────┐
│ 訪問回数を入力  │
├─────────────────┤
│ [10]            │ ← テキストボックス
├─────────────────┤
│ [OK] [キャンセル]│
└─────────────────┘
```

**バリデーション**:
- 数値のみ許可
- 1以上の整数
- 空白の場合はエラー

**エラーメッセージ**:
```
「1以上の数値を入力してください」
```

---

### 2.4 10回以上の訪問

#### 2.4.1 入力方法

**右クリックメニューの「直接入力」のみ**

**理由**:
- クリック連打で10回以上は不便
- 実運用では9回以上は稀

---

#### 2.4.2 表示

**セル内の表示**:
```
┌───┐
│10 │ ← 2桁
└───┘

┌───┐
│15 │ ← 2桁
└───┘
```

**セル幅の調整**:
- 2桁まで表示可能な幅を確保
- 3桁以上は想定しない（実運用で発生しない）

## 3. 重複の可否

### 3.1 重複パターンの整理

| 組み合わせ | 可否 | 備考 |
|-----------|------|------|
| 訪問 + 通い | ✅ 可能 | 通いの送り出しで職員が入室→訪問1件カウント |
| 訪問 + 入所日 | ✅ 可能 | 朝訪問、夕方入所 |
| 訪問 + 退所日 | ✅ 可能 | 朝退所、昼訪問 |
| 訪問 + 宿泊中 | △ 理論上可能 | 施設に宿泊中は訪問不要（実運用では稀） |

---

### 3.2 訪問 + 通い

**シナリオ**:
- 通いサービスの送り出し時に職員が自宅に入室
- 訪問1件としてカウント

**データ例**:
```
┌────┬───┐
│青柳│ ○ │ ← 通泊行
│美秋│ 1 │ ← 訪問行
└────┴───┘
```

**定員カウント**:
- 通い定員：前半+1、後半+1
- 訪問回数：+1

---

### 3.3 訪問 + 入所日

**シナリオ**:
- 朝に訪問サービス
- 夕方に入所

**データ例**:
```
┌────┬───┐
│青柳│入  │ ← 通泊行
│美秋│ 1 │ ← 訪問行
└────┴───┘
```

**定員カウント**:
- 通い定員：前半+1、後半+1
- 泊り定員：+1
- 訪問回数：+1

---

### 3.4 訪問 + 退所日

**シナリオ**:
- 朝に退所
- 昼に訪問サービス

**データ例**:
```
┌────┬───┐
│青柳│退  │ ← 通泊行
│美秋│ 1 │ ← 訪問行
└────┴───┘
```

**定員カウント**:
- 通い定員：前半+1、後半+1
- 泊り定員：+0（退所日は泊りカウントなし）
- 訪問回数：+1

---

### 3.5 訪問 + 宿泊中

**シナリオ**:
- 施設に宿泊中だが訪問サービスを記録
- 理論上は可能だが実運用では稀

**データ例**:
```
┌────┬───┬───┬───┐
│青柳│入  │ ○ │退  │ ← 通泊行
│美秋│   │ 1 │   │ ← 訪問行（宿泊中）
└────┴───┴───┴───┘
```

**定員カウント**:
- 通い定員：前半+1、後半+1
- 泊り定員：+1
- 訪問回数：+1

**実運用での扱い**:
- 通常は発生しない
- 発生した場合は特殊なケースとして記録

## 4. 宿泊期間の入力方法

### 4.1 ドラッグ操作（基本）

#### 4.1.1 通常の宿泊期間

**操作**:
```
5日から8日までドラッグ
  ↓
結果:
  5日 → 「入」
  6日 → 「○」（宿泊中）
  7日 → 「○」（宿泊中）
  8日 → 「退」
```

**視覚表現**:
```
┌────┬───┬───┬───┬───┐
│青柳│入  │ ○ │ ○ │退  │
│美秋│├青│├青│├青│├青│
└────┴───┴───┴───┴───┘
```

---

#### 4.1.2 ドラッグ中の視覚フィードバック

**マウスダウン（5日）**:
```
┌────┬───┬───┬───┬───┐
│青柳│[●]│   │   │   │ ← 5日が強調表示
│美秋│   │   │   │   │
└────┴───┴───┴───┴───┘
```

**ドラッグ中（5日→8日）**:
```
┌────┬───┬───┬───┬───┐
│青柳│[●]│[●]│[●]│[●]│ ← 5〜8日が強調表示
│美秋│   │   │   │   │
└────┴───┴───┴───┴───┘
```

**マウスアップ（8日）**:
```
┌────┬───┬───┬───┬───┐
│青柳│入  │ ○ │ ○ │退  │ ← 確定
│美秋│├青│├青│├青│├青│
└────┴───┴───┴───┴───┘
```

---

#### 4.1.3 ドラッグの方向

**順方向（左→右）**:
```
5日から8日までドラッグ
→ 5日「入」、6〜7日「○」、8日「退」
```

**逆方向（右→左）**:
```
8日から5日までドラッグ
→ 5日「入」、6〜7日「○」、8日「退」
（順方向と同じ結果）
```

**実装**:
```javascript
handleDragEnd(startCell, endCell) {
  // 開始日と終了日を正規化
  const startDate = new Date(startCell.date);
  const endDate = new Date(endCell.date);
  
  const checkInDate = startDate < endDate ? startDate : endDate;
  const checkOutDate = startDate < endDate ? endDate : startDate;
  
  // 入所〜退所を設定
  this.setStayPeriod(checkInDate, checkOutDate);
}
```

## 5. 削除⇔復元機能

### 5.1 基本仕様

#### 5.1.1 削除⇔復元の流れ

```
通常状態
  ↓ クリック
削除状態（グレー、点滅）
  ↓ 5秒以内にクリック
通常状態（復元）
  ↓ 5秒経過
完全削除（復元不可）
```

---

#### 5.1.2 復元タイムアウト

**設定値**:
- **5秒**（AppConfig.RESTORE_TIMEOUT = 5000）

**調整方法**:
```javascript
const AppConfig = {
  RESTORE_TIMEOUT: 5000,  // ミリ秒（調整可能）
  // ...
};
```

**将来的な拡張**:
- 設定画面で調整可能にする可能性あり
- 現時点では固定値

---

### 5.2 通常の通いセルの削除⇔復元

#### 5.2.1 状態遷移

**通常の通いセル（○）**:
```
○ → クリック → 削除状態（グレー、点滅）
  ↓ 5秒以内にクリック
○（復元）
  ↓ 5秒経過
空白（完全削除）
```

---

#### 5.2.2 削除状態の視覚表現

**CSS**:
```css
.cell-deleted {

@keyframes blink {
  0%, 100% { opacity: 1; }
  50% { opacity: 0.5; }
}
```

**表示例**:
```
┌────┬───┐
│青柳│[×]│ ← グレー、点滅
└────┴───┘
```

---

#### 5.2.3 ホバー時のツールチップ

**削除状態のセルにホバー**:

### 5.4 実装の詳細

#### 5.4.1 ScheduleCellのデータ構造

```javascript
class ScheduleCell {
  constructor(userId, date, cellType) {
    this.userId = userId;
    this.date = date;
    this.cellType = cellType;
    this.inputValue = "";
    this.note = "";
    this.actualFlags = {
      day: false,
      stay: false,
      visit: 0,
      halfDayType: 'full'
    };
    
    // 削除⇔復元機能
    this.deletedValue = null;  // 削除前の値
    this.deletedAt = null;     // 削除時刻（ミリ秒）
  }
  
  // 削除
  delete() {
    if (this.inputValue !== "") {
      this.deletedValue = this.inputValue;
      this.deletedAt = Date.now();
      this.inputValue = "";
      Logger?.debug(`Cell deleted: ${this.userId} ${this.date}`);
    }
  }
  
  // 復元
  restore() {
    if (this.canRestore()) {
      this.inputValue = this.deletedValue;
      this.deletedValue = null;
      this.deletedAt = null;
      Logger?.info(`Cell restored: ${this.userId} ${this.date}`);
      return true;
    }
    return false;
  }
  
  // 復元可能判定
  canRestore() {
    if (this.deletedValue === null) return false;
    const elapsed = Date.now() - this.deletedAt;
    return elapsed < AppConfig.RESTORE_TIMEOUT;
  }
  
  // 削除状態のクリア（タイムアウト時）
  clearDeletedState() {
    this.deletedValue = null;
    this.deletedAt = null;
  }
}
```

#### 5.4.2 タイムアウト処理

**実装**:
```javascript
class CellEditor {
  constructor(app) {
    this.app = app;
    this.restoreTimers = new Map();  // userId_date → timerId
  }
  
  handleCellClick(cell) {
    // 削除状態のセルをクリック → 復元
    if (cell.canRestore()) {
      this.restoreCell(cell);
      return;
    }
    
    // 通常のセルをクリック → 削除
    if (cell.inputValue !== "") {
      this.deleteCell(cell);
      return;
    }
    
    // 空白のセルをクリック → 通い（○）
    cell.inputValue = AppConfig.SYMBOLS.FULL_DAY;
    this.app.controllers.schedule.updateCell(cell);
  }
  
  deleteCell(cell) {
    cell.delete();
    this.app.controllers.schedule.updateCell(cell);
    
    // タイムアウトタイマーを設定
    const key = `${cell.userId}_${cell.date}`;
    const timerId = setTimeout(() => {
      cell.clearDeletedState();
      this.app.controllers.schedule.updateCell(cell);
      this.restoreTimers.delete(key);
      Logger?.info(`Cell restore timeout: ${key}`);
    }, AppConfig.RESTORE_TIMEOUT);
    
    this.restoreTimers.set(key, timerId);
  }
  
  restoreCell(cell) {
    cell.restore();
    this.app.controllers.schedule.updateCell(cell);
    
    // タイムアウトタイマーをキャンセル
    const key = `${cell.userId}_${cell.date}`;
    const timerId = this.restoreTimers.get(key);
    if (timerId) {
      clearTimeout(timerId);
      this.restoreTimers.delete(key);
    }
  }
}
```

### 5.5 画面を閉じた場合の動作

**仕様**:
- 画面を閉じる（ブラウザを閉じる、タブを閉じる）
- → 削除状態はクリア（復元不可）

**理由**:
- LocalStorageにdeletedValueを保存しない
- メモリ上のみで管理

**実装**:
```javascript
// LocalStorageに保存する際、削除状態は除外
toJSON() {
  return {
    userId: this.userId,
    date: this.date,
    cellType: this.cellType,
    inputValue: this.inputValue,
    note: this.note,
    actualFlags: this.actualFlags,
    // deletedValue, deletedAt は保存しない
  };
}
```
