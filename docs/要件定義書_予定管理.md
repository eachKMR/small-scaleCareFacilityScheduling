## 1. 通泊行の記号定義

### 1.1 基本記号

| 記号 | Unicode | 意味 | 定員カウント | クリック操作 |
|------|---------|------|-------------|-------------|
| **○** | U+25CB | 通い全日（通いのみ） | 通い: 前半1 + 後半1, 泊り: 0 | ✅ 可能 |
| **◎** | U+25CE | 連泊中（通い+泊り） | 通い: 前半1 + 後半1, 泊り: 1 | ❌ 不可 |
| **◓** | U+25D3 | 前半通い | 通い: 前半1 + 後半0, 泊り: 0 | ✅ 可能 |
| **◒** | U+25D2 | 後半通い | 通い: 前半0 + 後半1, 泊り: 0 | ✅ 可能 |
| **入** | - | 入所（宿泊開始） | 通い: 前半1 + 後半1, 泊り: 1 | ✅ 可能 |
| **退** | - | 退所（宿泊終了） | 通い: 前半1 + 後半1, 泊り: 0 | ✅ 可能 |

**重要な設計原則**:
- **クリック可能 = 操作可能**: 空欄、○、◓、◒、入、退
- **クリック不可 = 操作不要**: ◎（宿泊期間の中間日）
- **直感的な理解**: クリックできるかどうかで操作可否を判断

---

### 1.2 記号の詳細説明

#### 1.2.1 通い全日（○）

**意味**:
- 通いサービスのみを全日利用
- 前半（午前）と後半（午後）の両方を利用
- **泊りにはカウントされない**

**定員カウント**:
- 通い定員：前半+1、後半+1
- 泊り定員：+0

**使用頻度**: 最も一般的

**入力方法**:
- **空欄をクリック** → フォーカス + ○
- **○をクリック** → フォーカス + 空欄
- **シンプルなトグル**: 空欄 ⇔ ○
- 右クリックメニュー

---

#### 1.2.2 連泊中（◎）

**意味**:
- 宿泊期間の中間日（入と退の間）
- 通いと泊りの両方にカウントされる
- **手動では入力不可**（2ステップクリック時に自動設定）
- **クリック操作不可**

**定員カウント**:
- 通い定員：前半+1、後半+1
- 泊り定員：+1

**入力・操作方法**:
- **右クリック→「泊り」→退所日クリックで自動設定**
- **クリック不可**: クリックしても無反応
- **フォーカス不可**: フォーカスもできない
- **削除方法**: 宿泊期間全体を右クリック→クリア

**設計意図**:
- ◎は「中間状態」で、ユーザーが直接操作する必要がない
- クリック不可にすることで、UI設計がシンプルになる
- 誤操作を防止

---

#### 1.2.3 前半通い（◓）

**意味**: 通いサービスを前半（午前）のみ利用

**定員カウント**:
- 通い定員：前半+1、後半+0
- 泊り定員：+0

**入力方法**: 右クリックメニューのみ

---

#### 1.2.4 後半通い（◒）

**意味**: 通いサービスを後半（午後）のみ利用

**定員カウント**:
- 通い定員：前半+0、後半+1
- 泊り定員：+0

**入力方法**: 右クリックメニューのみ

---

#### 1.2.5 入所（入）

**意味**: 宿泊サービスの開始日

**定員カウント**:
- 通い定員：前半+1、後半+1
- 泊り定員：+1

**入力方法**: 右クリック→「泊り」の2ステップクリック方式で自動設定

**視覚表現**:
- 文字：「入」
- 背景色：薄青（#d0e8f0）
- 左罫線：青（#0066cc）、太さ3px

---

#### 1.2.6 退所（退）

**意味**: 宿泊サービスの終了日

**定員カウント**:
- 通い定員：前半+1、後半+1
- **泊り定員：+0**（退所日は泊りカウントしない）

**理由**: 退所日は朝に退所するため、その日の夜は宿泊しない

**入力方法**: 右クリック→「泊り」→退所日をクリックで自動設定

**視覚表現**:
- 文字：「退」
- 背景色：薄青（#d0e8f0）
- 右罫線：青（#0066cc）、太さ3px

## 2. 訪問回数の入力

### 2.1 訪問回数の表現

**表示形式**:
- 0回：空欄
- 1〜3回：数字（1、2、3）
- 4回以上：数字（4、5、6、...）

### 2.2 訪問回数の入力方法

#### 2.2.1 クリック連打

**動作**:
```
空欄 → 1 → 2 → 3 → 空欄 → 1 → ...
```

#### 2.2.2 右クリックメニュー

**メニュー項目**:
- 1回
- 2回
- 3回
- 直接入力（4回以上）
- クリア

## 3. 重複の可否

### 3.1 通泊行と訪問行の重複

**許可される重複**: すべて許可
- 訪問 + 通い（○）
- 訪問 + 入所（入）
- 訪問 + 退所（退）
- 訪問 + 宿泊中（◎）

## 4. 宿泊期間の入力方法

### 4.1 新規宿泊期間の作成（2ステップクリック方式）

#### 4.1.1 基本フロー

**ステップ1: 右クリック→「泊り」を選択**
```
1. 入所日にしたいセルを右クリック
2. メニューから「泊り」を選択
3. そのセルが「入」になる
4. ステータスバー表示: ⏱️ 退所日をクリックしてください（Escでキャンセル）
5. 入所日セルが点滅（アニメーション）
```

**ステップ2: 退所日をクリック**
```
1. 同じ利用者の右側のセルをクリック
2. 入所日〜退所日の期間が自動設定される
   - 入所日: 「入」
   - 中間日: 「◎」（複数日の場合）
   - 退所日: 「退」
3. ステータスバー非表示
4. 完了
```

---

#### 4.1.2 実装例

**最短宿泊（1泊2日）**:
```
Before:
┌───┬───┐
│   │   │
└───┴───┘
1日  2日

操作:
1. 1日を右クリック→「泊り」
2. 2日をクリック

After:
┌───┬───┐
│入  │退  │
└───┴───┘
1日  2日
```

**3泊4日**:
```
Before:
┌───┬───┬───┬───┐
│   │   │   │   │
└───┴───┴───┴───┘
5日  6日  7日  8日

操作:
1. 5日を右クリック→「泊り」
2. 8日をクリック

After:
┌───┬───┬───┬───┐
│入  │ ◎ │ ◎ │退  │
└───┴───┴───┴───┘
5日  6日  7日  8日
```

---

#### 4.1.3 キャンセル操作

**Escキーでキャンセル**:
```
1. 入所日を設定した状態
2. Escキーを押す
3. 「入」が空欄に戻る
4. ステータスバー非表示
```

**別のセルを右クリック**:
```
1. 入所日を設定した状態
2. 別のセルを右クリック→「泊り」
3. 前の入所日が空欄に戻る
4. 新しい入所日が設定される
```

---

#### 4.1.4 エラーケース ★v1.3.1で更新

**無効なセルをクリックした場合**:

**ケース1: 左側のセルをクリック**
```
ステータスバー表示（赤背景）:
❌ 退所日は入所日より後の日付を選択してください
```

**ケース2: 別の利用者のセルをクリック**
```
ステータスバー表示（赤背景）:
❌ 同じ利用者のセルを選択してください
```

**ケース3: 訪問行のセルをクリック**
```
ステータスバー表示（赤背景）:
❌ 通泊行のセルを選択してください
```

---

### 4.2 既存宿泊期間の削除

#### 4.2.1 右クリック→クリア

**操作**:
```
1. 宿泊期間中（入、◎、退）のいずれかを右クリック
2. メニューから「クリア」を選択
3. 期間全体が空欄になる
```

**例**:
```
Before: [入][◎][◎][退]
        1日 2日 3日 4日

操作: 2日(◎)を右クリック→「クリア」

After: [空][空][空][空]
      1日 2日 3日 4日
```

---

### 4.3 宿泊期間の調整

#### 4.3.1 期間の再作成

**方法**: 削除→新規作成

**手順**:
```
1. 既存の期間を右クリック→「クリア」
2. 新しい入所日を右クリック→「泊り」
3. 新しい退所日をクリック
```

---

### 4.4 ドラッグ操作の廃止

**廃止した機能**:
- ❌ 空欄からのドラッグによる宿泊期間作成
- ❌ 入をドラッグして入所日調整
- ❌ 退をドラッグして退所日調整
- ❌ 期間全体の移動ドラッグ

**廃止の理由**:
- 実装難易度が非常に高い
- イベント競合の制御が複雑
- ユーザー意図の曖昧性
- 使用頻度が低い
- 実装コスト >> UX向上効果

**代替手段**:
- 2ステップクリック方式（シンプル・明確・予測可能）
- 削除→再作成（3〜6回のクリックで完結）

## 5. 操作方法の詳細

### 5.1 通泊行の操作

| セル状態 | クリック可否 | クリック動作 | 右クリック |
|---------|------------|------------|-----------|
| **空欄** | ✅ 可能 | フォーカス + ○ | メニュー表示 |
| **○** | ✅ 可能 | フォーカス + 空欄 | メニュー表示 |
| **◓/◒** | ✅ 可能 | フォーカス + 空欄 | メニュー表示 |
| **入** | ✅ 可能 | フォーカスのみ | メニュー表示 |
| **退** | ✅ 可能 | フォーカスのみ | メニュー表示 |
| **◎** | ❌ 不可 | 無反応 | メニュー表示 |

**重要な設計原則**:
1. **空欄のクリック**: フォーカス + ○追加
2. **○のクリック**: フォーカス + 空欄に戻す（シンプルなトグル）
3. **宿泊期間作成**: 右クリック→「泊り」→退所日クリック
4. **◎はクリック不可**: 操作不要な中間状態
5. **宿泊期間削除**: 右クリック→クリアのみ
6. **ドラッグ操作**: 全面的に廃止

---

### 5.2 訪問行の操作

| 操作 | 動作 |
|------|------|
| **クリック** | フォーカス + 回数+1（0→1→2→3→0） |
| **右クリック** | フォーカス + メニュー表示 |

## 📝 実装時の注意事項

### 1. 2ステップクリック方式
- 状態管理: `waitingForCheckOut` フラグ
- ステータスバー表示の制御
- エラーハンドリング（無効なセル選択）
- キャンセル機能（Escキー、別の入所日設定）

### 2. シンプルなトグル
- 空欄 ⇔ ○ のシンプルな切り替え
- 削除状態という概念なし
- タイマー管理不要
- 訪問行との一貫性

### 3. 記号の扱い
- Unicode記号の正しい表示
- フォントによる表示の違い

### 4. フォーカス管理
- ◎はフォーカス不可
- フォーカスの視覚的明確性

### 5. クリック可否判定
- ◎のクリックイベント無効化
- カーソル表示の切り替え（pointer / default）

### 6. ドラッグ操作の削除
- ドラッグ関連のコードはすべて削除
- イベントリスナーも不要
