### 1.5 スクロール同期（Phase 1-A実装確定）

#### 1.5.1 要件

**課題**:
- 定員状況ヘッダーと予定グリッドは別コンテナ
- 横スクロールが独立していると、日付列がずれて見づらい

**解決策**:
- 定員状況ヘッダーと予定グリッドの**横スクロールを同期**
- どちらをスクロールしても、もう一方も追従する

---

#### 1.5.2 実装方法

**JavaScriptでスクロールイベントをバインド**:

```javascript
setupScrollSync(capacityContainer, gridContainer) {
    let isCapacityScrolling = false;
    let isGridScrolling = false;
    
    // 定員ヘッダーのスクロールをグリッドに反映
    capacityContainer.addEventListener('scroll', () => {
        if (isGridScrolling) {
            isGridScrolling = false;
            return;
        }
        isCapacityScrolling = true;
        gridContainer.scrollLeft = capacityContainer.scrollLeft;
    });
    
    // グリッドのスクロールを定員ヘッダーに反映
    gridContainer.addEventListener('scroll', () => {
        if (isCapacityScrolling) {
            isCapacityScrolling = false;
            return;
        }
        isGridScrolling = true;
        capacityContainer.scrollLeft = gridContainer.scrollLeft;
    });
}
```

**フラグの役割**:
- 無限ループを防止
- 一方のスクロールイベントが他方のスクロールを引き起こすのを制御

---

#### 1.5.3 CSS要件

**両方のコンテナに`overflow-x: auto`を指定**:

```css

    overflow-x: auto;
    overflow-y: hidden;
}

#schedule-grid {

    overflow-y: auto;
}

/* 両方のテーブルをコンテンツ幅に合わせる */

    width: max-content;
}
```

#### 1.6.3 空セル対策

**課題**:
訪問回数が0の行（空セル）では高さが維持されない

**解決策**:
CSS擬似要素で不可視のコンテンツを挿入

```css
.schedule-cell:empty::after {
    content: '\00a0'; /* non-breaking space */
    visibility: hidden;
}
```

**効果**:
- 空セルでも18pxの高さを確保
- レイアウトの崩れを防止

## 3. エラーハンドリング

### 3.1 エラーメッセージの表示方法

#### 3.1.1 トースト通知

**用途**:
- 一時的な情報
- 成功メッセージ
- 軽微なエラー

**表示位置**:
- 画面右下
- 自動的に消える（3秒後）

**実装**:
```javascript
showToast(message, type) {
  const toast = document.createElement('div');
  toast.className = `toast toast-${type}`;
  toast.textContent = message;
  
  document.body.appendChild(toast);
  
  setTimeout(() => {
    toast.classList.add('show');
  }, 10);
  
  setTimeout(() => {
    toast.classList.remove('show');
    setTimeout(() => {
      document.body.removeChild(toast);
    }, 300);
  }, 3000);
}
```

#### 3.1.2 モーダルダイアログ

**用途**:
- 重要なエラー
- 確認が必要な操作
- ユーザーの選択が必要

**表示位置**:
- 画面中央
- 背景をオーバーレイ

**実装**:
```javascript
showDialog(title, message, buttons) {
  return new Promise((resolve) => {
    const dialog = document.createElement('div');
    dialog.className = 'modal-overlay';
    dialog.innerHTML = `
      <div class="modal-dialog">
        <h3>${title}</h3>
        <p>${message}</p>
        <div class="modal-buttons">
          ${buttons.map(btn => `
            <button class="modal-button" data-value="${btn.value}">
              ${btn.label}
            </button>
          `).join('')}
        </div>
      </div>
    `;
    
    dialog.addEventListener('click', (e) => {
      if (e.target.classList.contains('modal-button')) {
        const value = e.target.dataset.value;
        document.body.removeChild(dialog);
        resolve(value);
      }
    });
    
    document.body.appendChild(dialog);
  });
}
```
