## 1. 画面レイアウト

### 1.1 全体構成

#### 1.1.1 画面構成

```
┌─────────────────────────────────────────┐
│ツールバー（5%）                          │
│[2025年11月▼] [作業内容を取り込む]       │
│[作業内容を保存] [算定一覧を取り込む] [⚙️]│
├─────────────────────────────────────────┤
│定員状況ヘッダー（10%）- 固定             │
│    │1金│2土│3日│...                    │
│通い│○13│○14│×16│                       │
│泊り│○7 │○8 │△9 │                       │
│訪問│ 5 │ 8 │ 3 │                       │
╞════╪═══╪═══╪═══╡← 太罫線               │
│予定グリッド（85%）- スクロール           │
│ 29名 × 2行 = 58行                       │
│  - 1行目: 通泊行（統合）                │
│  - 2行目: 訪問行                        │
└─────────────────────────────────────────┘
```

#### 1.1.2 画面占有率

| 要素 | 割合 | 備考 |
|------|------|------|
| ツールバー | 5% | 固定 |
| 定員状況ヘッダー | 10% | 固定、横スクロール連動 |
| 予定グリッド | 85% | 縦横スクロール可能 |

**Phase 1-A改善結果**:
- 予定グリッド占有率: 65% → 85%（+20%向上）
- 表示行数: 約10名 → 約20名（2倍）

### 1.4 予定グリッド

#### 1.4.1 構造（33列 × 58行）

**列構成**:
```
列1: 利用者名（固定列）
列2: 前月（左特別セル）
列3-33: 1日〜31日
列34: 翌月（右特別セル）
```

**行構成**:
```
29名 × 2行 = 58行
  - 奇数行（1, 3, 5, ...）: 通泊行
  - 偶数行（2, 4, 6, ...）: 訪問行
```

---

#### 1.4.4 セルのスタイル

**通常セル**:
```css
.schedule-cell {
  width: 32px;
  min-width: 32px;
  max-width: 32px;
  text-align: center;
  vertical-align: middle;
  border: 1px solid #ddd;
  cursor: pointer;
  box-sizing: border-box;
}

.schedule-cell:hover {
  background: #f5f5f5;
}
```

**フォーカス中のセル** ★v1.2で追加:
```css
/**
 * フォーカス中のセル
 * - 青枠で強調表示
 * - 外側にぼかしを追加
 */
.schedule-cell:focus,
.schedule-cell.focused {
  outline: 2px solid #0066cc;  /* 青枠 */
  outline-offset: -2px;
  box-shadow: 0 0 0 3px rgba(0, 102, 204, 0.2);  /* 外側にぼかし */
  z-index: 10;
  position: relative;
}

/* フォーカス時のホバー効果は無効化 */
.schedule-cell.focused:hover {
  background: transparent;
}
```

**視覚的表示例**:
```
通常: 
┌───┬───┬───┐
│ ○ │   │ 1 │
└───┴───┴───┘

フォーカス中:
┌───┬═══┬───┐  ← 太い青枠 + ぼかし
│ ○ ║ ○ ║ 1 │
└───┴═══┴───┘
```

**特別セル**:
```css
.cell-special {
  background-color: #f5f5f5;  /* 灰背景 */
  border: 1px dashed #999;    /* 点線枠 */
}
```

**宿泊期間**:

### 1.5 スクロール同期（Phase 1-A実装確定）

#### 1.5.1 要件

**課題**:
- 定員状況ヘッダーと予定グリッドは別コンテナ
- 横スクロールが独立していると、日付列がずれて見づらい

**解決策**:
- 定員状況ヘッダーと予定グリッドの**横スクロールを同期**
- どちらをスクロールしても、もう一方も追従する

## 5. UI仕様（レイアウト、操作方法）

### 5.1 操作方法のまとめ ★v1.2で更新

**通泊行**:

| 操作 | 動作 |
|------|------|
| ホバー | ツールチップ表示 |
| 1回目クリック | フォーカス（青枠表示） ★v1.2 |
| フォーカス中に2回目クリック | 削除⇔復元、または○追加 ★v1.2 |
| フォーカス中にドラッグ | 入所→退所の範囲選択 ★v1.2 |
| 右クリック | フォーカス + メニュー表示 ★v1.2 |

**訪問行**:

| 操作 | 動作 |
|------|------|
| クリック | フォーカス + 回数+1（0→1→2→3→0） ★v1.2で変更 |
| 右クリック | フォーカス + メニュー表示 ★v1.2 |

**詳細は [要件定義書_予定管理_v1.1.md](要件定義書_予定管理_v1.1.md) を参照**

### 5.4 アクセシビリティ

#### 5.4.1 キーボード操作（将来的に実装）

**セルの移動**:

| キー | 動作 |
|------|------|
| Tab | 次のセル |
| Shift+Tab | 前のセル |
| ↑ | 上のセル |
| ↓ | 下のセル |
| ← | 左のセル |
| → | 右のセル |

**編集操作**:

| キー | 動作 |
|------|------|
| Enter | 編集モード開始 |
| Esc | 編集モードキャンセル |
| Delete | セルをクリア |
| 数字キー | 訪問行：直接入力 |

**ショートカットキー**:

| キー | 動作 |
|------|------|
| Ctrl+S | 保存 |
| Ctrl+Z | 元に戻す（未実装） |
| Ctrl+Y | やり直し（未実装） |
