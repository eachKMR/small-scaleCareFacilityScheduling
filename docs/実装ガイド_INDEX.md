### 1. 実装ガイド（最優先）★★★
- 実装すべき内容はここに全て書かれている
- コード例、テスト項目も含む
- **迷ったらまずここを見る**

### 2. 未確定項目リスト（参考）★★
- 実装中に判断が必要な項目
- 推奨案を参考にできる
- **実装ガイドに書いていないことで迷ったら見る**

### 3. 要件定義書（詳細確認）★
- 背景や詳細仕様を知りたい時
- **より深く理解したい時に見る**

```
実装中に迷った
```

## 💡 このINDEXの使い方（クイックリファレンス）

### 実装担当者（GitHub Copilot）向け

**Q. 今何をすべき？**  
→ 「🔄 実装中」のフェーズの実装ガイドを開く

**Q. このフェーズは何を作るの？**  
→ 各フェーズの「実装内容」「実装ファイル」を確認

**Q. 完了したかどうかの判断基準は？**  
→ 「完了条件」のチェックボックスを確認

**Q. 次は何をするの？**  
→ 「次のステップ」を確認、設計担当に報告

### 設計担当者（Claude）向け

**Q. 次の実装ガイドを作るタイミングは？**  
→ 現在のフェーズ完了報告を受けた後

**Q. 優先順位の判断基準は？**  
→ ユーザー価値、依存関係、実装難易度を考慮

**Q. フェーズの分割基準は？**  
→ 1フェーズ = 4〜6時間、機能的にまとまりがある単位
