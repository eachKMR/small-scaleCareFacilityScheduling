## 📖 このドキュメントについて

このドキュメントは、**小規模多機能型居宅介護における調整業務の制約**、特に**ケアマネージャー・調整担当者の視点からのリソース管理と稼働率の最適化**をまとめたものです。

### 対象読者

- プロジェクトBの設計担当者
- 調整支援機能を設計・実装する開発者
- 小規模多機能施設のケアマネージャー・調整担当者

### 読了後に理解できること

- 3サービス（通い・訪問・泊まり）の同時管理の難しさ
- 定員制約下での稼働率最適化の重要性
- ケアプラン変更の複雑さ
- 制度的制約が調整業務に与える影響
- プロジェクトBでサポートすべき調整業務

---

## 1. 調整業務の本質

### 1.1 ケアマネージャー・調整担当者の役割

小規模多機能型居宅介護では、ケアマネージャーが**3つのサービスを一体的に管理**します。

```
ケアマネージャーの役割：
  ├─ ケアプランの作成・変更
  ├─ 利用者・家族との相談対応
  ├─ 3サービス（通い・訪問・泊まり）の調整
  ├─ モニタリングと見直し
  ├─ 給付管理
  └─ 多職種・外部機関との連携
```

**重要**: 介護職員を兼務することも多い（計画作成業務は週24時間程度を想定）

---

### 1.2 調整業務の視点

調整業務は、**リソース管理**と**稼働率の最適化**が中心です。

| 視点 | 内容 |
|------|------|
| **定員の有効活用** | 通い15人、泊まり9人を埋める |
| **利用者のニーズ** | 柔軟なサービス提供 |
| **稼働率** | 登録定員29人の確保と維持 |
| **収益性** | 月額定額制での採算確保 |

---

## 2. 3サービスの同時管理

### 2.1 通い・訪問・泊まりのバランス

**課題**: 3つのサービスを同時に調整する必要がある

```
例：ある利用者の1週間

月曜：通い（午前のみ）
火曜：訪問（朝・夕）
水曜：通い（終日）
木曜：訪問（朝）+ 通い（午後）
金曜：泊まり（入所）
土曜：泊まり（継続）
日曜：泊まり（退所）
```

**調整担当者の作業**:
1. 月曜の通い定員を確認（前半のみ使用）
2. 火曜の訪問スケジュールに組み込む
3. 水曜の通い定員を確認（終日使用）
4. 木曜の訪問と通いの両方を調整
5. 金曜〜日曜の泊まり定員を確保
6. 居室の割当
7. ケアプランに記載
8. 家族・本人に説明

---

### 2.2 定員の制約

3つのサービスそれぞれに定員があります。

| サービス | 定員 | 管理単位 |
|---------|------|---------|
| **登録** | 29人 | 事業所全体 |
| **通い** | 15人 | 1日あたり（前半・後半別） |
| **泊まり** | 9人 | 1日あたり |
| **訪問** | なし | 制限なし |

**定員の詳細な法的枠組みについては、L0_業務_定員の法的枠組み.md を参照してください。**

**調整の難しさ**:
```
通いが埋まっている日：
→ 訪問で対応できないか？
→ 別の日に通いをずらせないか？

泊まりが埋まっている日：
→ 通いと訪問で代替できないか？
→ 他の利用者の泊まりを調整できないか？
```

---

### 2.3 柔軟性と複雑性のトレードオフ

#### メリット：柔軟性

小規模多機能の最大の特徴は**柔軟性**です。

**事例1**: 家族の都合で急に泊まりが必要
```
当日の朝：
家族「今夜、泊まれますか？」
  ↓
調整担当者：
1. 泊まり定員を確認（空きがある）
2. 通いから泊まりに変更
3. 居室を割当
4. 夕食・朝食を手配
5. ケアプランを変更
6. 給付管理を更新
```

**事例2**: 体調不良で訪問から通いに変更
```
前日の夕方：
家族「明日は通いに変更したい」
  ↓
調整担当者：
1. 通い定員を確認（空きがある）
2. 訪問スケジュールから削除
3. 通いスケジュールに追加
4. 送迎の手配
5. ケアプランを変更
```

---

#### デメリット：複雑性

柔軟性の裏返しとして、**ケアプラン変更が頻繁**になります。

**ケアマネの声**（求人サイト等より）:

> 「利用者さんの状況に合わせて柔軟にサービス内容を変える必要がある」
> 
> 「ケアプラン変更に関わる作業が複雑になる点は大変」
> 
> 「3つのサービスを管理するので、介護支援のスキルが問われる」

---

## 3. 定員制約下での稼働率最適化

### 3.1 登録率の課題

**現実**: 登録定員29人に対して、実際の登録者数は少ない傾向にあります。

**主な原因**:
1. 認知度不足
2. ケアマネージャー変更のハードル
3. 他サービス併用不可
4. 泊まり定員が埋まりやすい

**詳細な登録率データと新規利用者獲得の戦略については、L0_業務_新規利用者獲得.md を参照してください。**

---

### 3.2 稼働率の重要性

小規模多機能は**月額定額制**です。

**収支構造**:
```
収入：登録者数 × 月額単位数 × 単価
支出：人件費 + 施設費 + 運営費

→ 登録者数が少ないと採算が合わない
```

**調査結果**（令和3年度）:
- 収支差率：4.7%（全サービス平均3.0%）
- 金額ベース：24.5万円/月

**つまり**: 
- 登録者数を確保しないと経営が厳しい
- 定員29人の80%以上（約23人）が理想

---

### 3.3 新規利用者獲得の難しさ

**問い合わせ対応の例**:

```
相談者：「母が通いと泊まりを利用したい」
調整担当者：
1. 登録定員の空きを確認（OK）
2. 通いの空き状況を確認（週2〜3回は可能）
3. 泊まりの空き状況を確認（週末は埋まりがち）
  ↓
「通いは大丈夫ですが、泊まりは週末が難しいです」
  ↓
相談者：「週末に泊まれないなら困る」→ 利用見送り
```

**課題**: 
- 泊まりが埋まっていることが利用のネックになる
- 実際には特定の日だけ埋まっていることが多い
- 月全体の空き状況が把握できていない

---

## 4. ケアプラン変更の複雑さ

### 4.1 頻繁な変更

**小規模多機能の特徴**: 利用者の状況に応じて**いつでも**サービスを変更できる

**変更の頻度**:
```
週単位：
- 利用者A：今週は通い3回 → 来週は通い2回+泊まり1回
- 利用者B：月曜の通いをキャンセル → 火曜の訪問に変更

月単位：
- 季節の変わり目に多数の利用者が変更
- 家族の都合（仕事、介護休暇等）で変更
```

---

### 4.2 変更作業の内容

**1回の変更で必要な作業**:

```
1. 利用者・家族からの相談
   ↓
2. 定員の確認（通い・泊まり）
   ↓
3. スケジュールの調整
   ↓
4. ケアプランの変更（文書作成）
   ↓
5. 利用者・家族への説明・同意
   ↓
6. 職員への共有
   ↓
7. 給付管理の更新
   ↓
8. モニタリング
```

**所要時間**: 1回あたり30分〜1時間

**1ヶ月の変更回数**: 登録者29人 × 平均1〜2回 = **30〜60回**

---

### 4.3 ケアマネの業務負担

**ケアマネの1週間**（典型例）:

```
計画作成業務：週24時間
  ├─ ケアプラン作成・変更：10時間
  ├─ モニタリング：5時間
  ├─ 相談対応：4時間
  ├─ 給付管理：3時間
  └─ 多職種連携：2時間

介護業務：週16時間（兼務）
  ├─ 通いサービス：8時間
  ├─ 訪問サービス：4時間
  └─ その他：4時間

合計：週40時間
```

**課題**: 
- 計画作成業務と介護業務の両立
- 変更対応に追われる
- ゆっくりプランを練る時間がない

---

## 5. 制度的制約

### 5.1 ケアマネージャーの専属制

小規模多機能を利用する場合、**事業所専属のケアマネに変更**する必要があります。

#### 利用開始時の手続き

```
1. 居宅ケアマネとの契約解除
   ↓
2. 小規模多機能のケアマネと契約
   ↓
3. ケアプラン作成依頼届出書の提出
   ↓
4. 新しいケアプランの作成
```

#### 利用者・家族の心理的ハードル

> 「今のケアマネさんは気心知れているから変えたくない」
> 
> 「新しいケアマネさんとうまくやれるか不安」

**結果**: 小規模多機能の利用を躊躇する要因の一つ

---

### 5.2 他サービスとの併用制限

小規模多機能を利用すると、**多くのサービスが併用できなくなります**。

**調整業務への影響**:

```
相談者：「デイサービスは続けたい」
  ↓
調整担当者：
「小規模多機能では、通いサービスがデイサービスに
 相当するため、外部のデイサービスは利用できません」
  ↓
相談者：「それなら小規模多機能は利用できない」
```

**課題**: サービス併用制限が利用のハードルになる

**併用可能・不可のサービスの詳細については、L0_業務_小規模多機能とは.md の「6. 併用可能なサービス」を参照してください。**

---

### 5.3 給付管理の複雑さ

#### 月途中の利用開始・終了

**ケース1**: 月の途中から利用開始

```
11月15日から小規模多機能を利用開始

11月1日〜14日：居宅ケアマネが給付管理
11月15日〜30日：小規模多機能のケアマネが給付管理

→ 給付管理票は居宅ケアマネが作成
→ 居宅介護支援費は算定可能
```

**ケース2**: 月の途中で終了

```
11月20日まで小規模多機能を利用

11月1日〜20日：小規模多機能のケアマネが給付管理
11月21日〜30日：居宅ケアマネが給付管理

→ 給付管理票は居宅ケアマネが作成
→ 居宅介護支援費は算定可能
```

#### 調整業務への影響

- 月途中の利用開始・終了時は、両方のケアマネと連携が必要
- 給付管理票の作成責任を明確にする
- サービス利用の届出書類の提出

**課題**: 事務手続きが煩雑

---

## 6. 職員不足の中での調整業務

### 6.1 職員の充足状況

厚生労働省の調査（平成27年）:

```
職員の充足状況：
  「まったく足りない」  :  7.8%
  「あまり足りていない」: 42.7%
                   合計： 50.5%
```

**職員の平均人数**:
- 実人数：15.32人
- 常勤換算：11.28人

**登録定員29人に対して**: 
- 1人の職員で約2.6人の利用者を担当
- ギリギリの人員配置

---

### 6.2 業務負担への影響

**職員不足の影響**:

```
人員配置がギリギリ
  ↓
・ケアマネが介護業務も兼務
・残業が増える
・教育体制が不十分
・新人が育ちにくい
  ↓
・さらに職員が辞める
・人手不足が加速
```

**調整業務への影響**:
- ケアプラン作成に時間が取れない
- 丁寧なモニタリングができない
- 利用者・家族との相談時間が短い
- 多職種連携が不十分

---

### 6.3 効率化の必要性

**現状**: 手作業による調整が多い

```
例：空き状況の確認
1. 紙の予定表を見る
2. 通いの人数を数える
3. 泊まりの人数を数える
4. 前半・後半を区別して数える
5. 電話で回答

→ 1件あたり5〜10分
```

**効率化のニーズ**:
- 空き状況の即座の把握
- 稼働率の可視化
- 予定変更の履歴管理
- ケアプラン変更の簡素化

---

## 7. プロジェクトBでサポートすべき調整業務

### 7.1 3サービスの稼働状況の可視化

**現状の課題**: 
- 通い・泊まりの空き状況が一目でわからない
- 月全体の稼働率が把握できない

**プロジェクトBでの対応**:

```
ダッシュボード表示：
  ┌─────────────────────┐
  │ 11月の稼働状況        │
  ├─────────────────────┤
  │ 登録者数：25/29人     │
  │ 登録率：86%           │
  ├─────────────────────┤
  │ 通い稼働率：78%       │
  │ 泊まり稼働率：65%     │
  └─────────────────────┘

日別の空き状況：
  11/25（月）通い：12/15人 泊まり：7/9人
  11/26（火）通い：14/15人 泊まり：9/9人 満室
  11/27（水）通い：10/15人 泊まり：6/9人
```

---

### 7.2 空き枠の最適活用

**現状の課題**: 
- 「泊まりが埋まっている」と断るケースが多い
- 実際には特定の日だけ埋まっている

**プロジェクトBでの対応**:

```
新規相談時の支援：
  ↓
「週末の泊まりを希望」
  ↓
システム：
「土曜は満室ですが、金曜と日曜は空いています」
  ↓
調整担当者：
「金曜・日曜なら可能です。ご検討いただけますか？」
```

---

### 7.3 予定変更の履歴管理

**現状の課題**: 
- 変更履歴が残らない
- 「いつ、誰が、何を変更したか」がわからない

**プロジェクトBでの対応**:

```
変更履歴の記録：
  11/20 10:30 利用者A 通い→泊まりに変更（担当：山田）
  11/21 14:15 利用者B 訪問追加（担当：田中）
  11/22 09:00 利用者C 泊まりキャンセル（担当：山田）

→ いつでも振り返り可能
→ トラブル時の確認
→ モニタリングの資料
```

---

### 7.4 定員チェックの自動化

**現状の課題**: 
- 予定を入れる度に手作業でカウント
- 定員オーバーに気づかないことがある

**プロジェクトBでの対応**:

```
予定入力時：
  ↓
システムが自動チェック：
「11/26（火）の通いは既に15人です」
  ↓
警告表示：
⚠️ 定員に達しています。
  別の日への変更をご検討ください。
```

---

### 7.5 利用者ごとのサービス利用パターンの把握

**現状の課題**: 
- 「この利用者は普段どのくらい利用しているか」がすぐわからない
- ケアプラン見直し時に過去のデータを探す

**プロジェクトBでの対応**:

```
利用者Aの利用パターン（11月）：
  ├─ 通い：12回（前半8回、後半4回）
  ├─ 訪問：8回
  └─ 泊まり：4泊

月別推移：
  10月：通い10回、訪問10回、泊まり2泊
  11月：通い12回、訪問 8回、泊まり4泊
  
→ 通いが増加、泊まりが増加傾向
→ 在宅生活の継続が難しくなっている可能性
```

## 8. まとめ

### 8.1 調整業務の本質（再掲）

```
ケアマネージャー・調整担当者の視点：
  ├─ 3サービスの同時管理
  ├─ 定員制約下での稼働率最適化
  ├─ 柔軟なサービス提供
  ├─ 頻繁なケアプラン変更
  └─ 職員不足の中での効率化
```

---

### 8.2 調整業務の制約

| 制約 | 内容 |
|------|------|
| **定員** | 通い15人、泊まり9人、登録29人 |
| **制度** | ケアマネ専属、併用不可サービス |
| **業務量** | ケアプラン変更が頻繁、介護業務も兼務 |
| **職員不足** | 50%が「足りない」と感じる |
| **稼働率** | 登録率が低い（40%未満が44.4%） |

---

### 8.3 プロジェクトBの役割

プロジェクトBは、**調整業務の効率化と稼働率の最適化**を支援します：

1. **3サービスの可視化** - 一目で全体を把握
2. **空き枠の最適活用** - 新規獲得を支援
3. **予定変更の効率化** - 履歴管理、自動チェック
4. **稼働率の向上** - 定員の有効活用
5. **業務負担の軽減** - 手作業の削減

**ゴール**:
調整担当者が、限られた人員で、定員を有効活用しながら、利用者のニーズに応えられるようにする。

**介護ソフト（CAREKARTE）との連携については、L0_業務_介護ソフトとの関係.md を参照してください。**
