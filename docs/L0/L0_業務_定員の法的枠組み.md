# L0_業務_定員の法的枠組み

**作成日**: 2025年11月21日  
**カテゴリ**: 第0層 - 業務知識  
**バージョン**: 1.1  
**根拠法令**: 指定地域密着型サービスの事業の人員、設備及び運営に関する基準（基準第66条、第82条）

---

## 📖 このドキュメントについて

このドキュメントは、**小規模多機能型居宅介護の定員に関する法的枠組み**をまとめたものです。

### 対象読者

- プロジェクトBの設計担当者
- 定員管理機能を実装する開発者
- 小規模多機能施設の調整担当者

### 読了後に理解できること

- 登録定員と利用定員の違い
- 通い・泊まりの定員の算出方法
- 定員超過が認められる条件
- 「同時利用者数」の正しい解釈
- 過少サービス減算の基準

---

## 1. 定員の種類

小規模多機能型居宅介護には、**3種類の定員**があります。

### 1.1 定員の全体像

```
登録定員（29人以下）
  ├─ 通いサービス利用定員（登録定員の1/2 〜 15人）
  └─ 宿泊サービス利用定員（通い定員の1/3 〜 9人）
```

| 定員の種類 | 意味 | 基準 |
|-----------|------|------|
| **登録定員** | 事業所に登録できる利用者の総数 | 29人以下 |
| **通いサービス利用定員** | 1日に同時に通いサービスを利用できる人数 | 登録定員の1/2 〜 15人 |
| **宿泊サービス利用定員** | 1日に同時に宿泊サービスを利用できる人数 | 通い定員の1/3 〜 9人 |

**重要**: 訪問サービスには利用定員の規定はありません。

---

## 2. 登録定員（基準第66条）

### 2.1 基本ルール

#### 上限

**29人以下**

- サテライト事業所の場合：**18人以下**

#### 根拠条文

> 指定小規模多機能型居宅介護事業所は、その登録定員を29人（サテライト事業所にあっては、18人）以下としなければならない
> 
> — 基準第66条第1項

---

### 2.2 重要な制約：複数事業所利用の禁止

利用者は**1か所の事業所にのみ登録**できます。

#### ルール

- ✅ A事業所に登録 → A事業所のサービスを利用
- ❌ A事業所とB事業所に同時登録 → **不可**
- ❌ A事業所を利用しながらB事業所も利用 → **不可**

#### 理由

> 利用者と従業者のなじみの関係を築きながらサービスを提供する観点から、利用者は１か所の指定小規模多機能型居宅介護事業所に限って利用者登録を行うことができる

**プロジェクトBへの影響**:
- 利用者マスタは事業所ごとに完全に独立
- 他事業所のデータとの連携は不要

---

## 3. 通いサービス利用定員（基準第66条第2項）

### 3.1 基本ルール

#### 算出方法

```
下限 = 登録定員 ÷ 2
上限 = 15人（または特例の場合16〜18人）
```

| 登録定員 | 通い定員の下限 | 通い定員の上限（標準） | 通い定員の上限（特例） |
|---------|--------------|---------------------|---------------------|
| 20人 | 10人 | 15人 | - |
| 25人 | 12.5人 → 13人 | 15人 | - |
| 26人 | 13人 | 15人 | 16人 |
| 27人 | 13.5人 → 14人 | 15人 | 16人 |
| 28人 | 14人 | 15人 | 17人 |
| 29人 | 14.5人 → 15人 | 15人 | 18人 |

**注意**: 下限で端数が出る場合は、切り上げまたは切り捨ての判断が必要（実務上は切り上げが一般的）

---

### 3.2 特例：通い定員の引き上げ（16人〜18人）

#### 条件

登録定員が**26人以上29人以下**の場合、以下の条件を満たせば通い定員を引き上げ可能。

| 登録定員 | 引き上げ後の通い定員 |
|---------|---------------------|
| 26人 | 16人 |
| 27人 | 16人 |
| 28人 | 17人 |
| 29人 | 18人 |

#### 必要な条件

1. **登録定員が26人以上であること**
2. **居間及び食堂の面積要件**
   - 利用者の処遇に支障がないと認められる十分な広さ
   - **一人当たり3㎡以上**

#### 面積の考え方（Q&A 1599より）

**原則**:
- 居間及び食堂として届け出たスペースの合計で計算

**例外**:
- 居間・食堂以外の部屋でも、日常的に一体的に利用可能な場所がある場合
- 「利用者の処遇に支障がない」と認められる場合
- これらの部屋を含めて「一人当たり3㎡以上」として計算可能

**計算例**:

```
通い定員18人に引き上げる場合：
必要面積 = 18人 × 3㎡ = 54㎡以上
```

---

### 3.3 サテライト事業所の場合

- **通い定員の上限**: 12人

---

## 4. 宿泊サービス利用定員（基準第66条第2項）

### 4.1 基本ルール

#### 算出方法

```
下限 = 通い定員 ÷ 3
上限 = 9人
```

| 通い定員 | 泊まり定員の下限 | 泊まり定員の上限 |
|---------|----------------|----------------|
| 12人 | 4人 | 9人 |
| 15人 | 5人 | 9人 |
| 16人 | 5.3人 → 6人 | 9人 |
| 18人 | 6人 | 9人 |

**注意**: 下限で端数が出る場合の扱いは実務上の判断が必要

---

### 4.2 サテライト事業所の場合

- **泊まり定員の上限**: 6人

**泊まりサービスの居室管理の重要性については、L0_業務_居室管理の重要性.md を参照してください。**

## 5. 利用定員の正しい解釈

### 5.1 「同時利用者数」とは

**非常に重要**: 利用定員は「延べ人数」ではなく「同時にサービス提供を受ける者の上限」です。

#### 根拠（Q&A 1559）

> この場合における利用定員については、当該指定小規模多機能型居宅介護事業所において１日当たりの同時にサービスの提供を受ける者の上限を指すものであり、１日当たりの延べ人数ではない

---

### 5.2 具体例：時間帯別の利用

#### 例1：午前・午後で利用者が入れ替わる

```
通い定員15人の事業所の場合

午前（9:00-12:00）
  ├─ 利用者A〜O（15人） ← 定員内

午後（13:00-17:00）
  ├─ 利用者P〜Y（10人） ← 定員内

同日の延べ人数 = 25人
最大同時利用者数 = 15人 ← これが定員判定の基準
```

**結論**: 午前15人、午後別の10人という使い方も**問題なし**

---

#### 例2：1日中利用する人と午前のみ・午後のみの人が混在

```
通い定員15人の事業所の場合

終日利用者: A〜J（10人）
午前のみ: K〜M（3人）
午後のみ: N〜P（3人）

午前の同時利用者数 = 10 + 3 = 13人 ← 定員内
午後の同時利用者数 = 10 + 3 = 13人 ← 定員内
```

**結論**: 延べ16人だが、同時には13人なので**問題なし**

---

### 5.3 プロジェクトBへの影響

この解釈は、プロジェクトBの設計に重要な影響を与えます。

#### 通いセクションの設計要件

- **前半・後半の定員を別々にカウント**する必要がある
- 前半のみ、後半のみ、終日の利用者を区別
- **1日の延べ人数ではなく、前半・後半それぞれの同時利用者数**を基準とする

#### データ構造への影響

```
通い予定データ
  ├─ 前半（午前）: 利用者リスト
  │   └─ 同時利用者数をカウント → 定員と比較
  └─ 後半（午後）: 利用者リスト
      └─ 同時利用者数をカウント → 定員と比較
```

## 6. 定員の弾力運用（基準第82条）

### 6.1 定員超過が認められる条件

通常は定員を守る必要がありますが、**特に必要と認められる場合**は定員超過も可能です。

#### 根拠

> 基準第82条の規定により、特に必要と認められる場合は、当該利用定員を超えるサービス提供も差し支えない

---

### 6.2 弾力運用の考え方

#### 柔軟な組み合わせ

> 指定小規模多機能型居宅介護が利用者の心身の状況に応じ、柔軟に通いサービス、訪問サービス、宿泊サービスを組み合わせて提供されるものであることを踏まえ、適切なサービス提供を行うこと

#### 想定される例

- 急な体調変化で宿泊が必要になった
- 家族の緊急事態で急遽通いが必要になった
- 災害等の特殊事情

**注意**: 恒常的な定員超過は認められません。

---

### 6.3 プロジェクトBでの扱い

#### 定員チェック機能

- 定員超過を**警告**として表示
- ただし、**エラーとして禁止はしない**
- 調整担当者が判断できるようにする

#### UI表示例

```
⚠️ 通い定員超過（16人/15人）
※特に必要と認められる場合は可能です
```

---

## 7. その他の重要事項

### 7.1 有料老人ホーム入居者の利用

**可能**:
- 併設している有料老人ホームの入居者が小規模多機能を利用することは可能

**注意**:
- 特定施設入居者生活介護を受けている間は、介護報酬は算定できない

---

### 7.2 養護老人ホーム入所者の利用

**不可**:
- 養護老人ホームの入所者は想定していない

**理由**:
- 養護老人ホームは措置費の下で施設サービスとして基礎的な生活支援が行われている

---

## 8. 定員の設定例

### 8.1 標準的な設定例

#### パターン1：登録25人の場合

```
登録定員: 25人
通い定員: 12人〜15人（実際には15人が多い）
泊まり定員: 5人〜9人（実際には9人が多い）
```

#### パターン2：登録29人の場合（標準）

```
登録定員: 29人
通い定員: 15人
泊まり定員: 9人
```

#### パターン3：登録29人の場合（特例）

```
登録定員: 29人
通い定員: 18人（居間・食堂54㎡以上確保）
泊まり定員: 9人
```

---

### 8.2 サテライト事業所の設定例

```
登録定員: 18人
通い定員: 12人
泊まり定員: 6人
```

### 9.2 データ検証ルール

#### 登録時の検証

```javascript
// 登録定員の上限チェック
if (登録利用者数 > 29) {
  警告("登録定員29人を超えています");
}
```

#### 通い予定入力時の検証

```javascript
// 前半の同時利用者数チェック
if (前半の同時利用者数 > 通い定員) {
  警告("通い定員を超えています（前半）");
}

// 後半の同時利用者数チェック
if (後半の同時利用者数 > 通い定員) {
  警告("通い定員を超えています（後半）");
}
```

#### 泊まり予定入力時の検証

```javascript
// 同時宿泊者数チェック
if (当日の宿泊者数 > 泊まり定員) {
  警告("泊まり定員を超えています");
}
```

## 10. 過少サービス減算

### 10.1 減算の基準

**登録者1人当たりの平均サービス提供回数が週4回に満たない場合、減算が適用されます。**

#### 減算率

```
所定単位数の100分の70
→ 実質30%の減算
```

**根拠**: 厚生労働省告示（令和6年度報酬改定）

---

### 10.2 計算方法

#### 平均サービス提供回数の算出

```
登録者1人当たり平均回数 = 
  (月間サービス提供回数合計 ÷ 当該月の日数 ÷ 登録者数) × 7
```

**例**: 登録者25人、11月（30日）の場合

```
月間提供回数合計: 500回
計算: (500 ÷ 30 ÷ 25) × 7 = 4.67回/週

→ 週4回以上なので、減算なし
```

---

### 10.3 サービス提供回数のカウント方法

| サービス | カウント方法 |
|---------|------------|
| **通いサービス** | 利用した回数（1日複数回利用でも複数回として算定） |
| **訪問サービス** | 利用した回数（見守り・声かけ等も含む） |
| **宿泊サービス** | 利用した泊数（通いに引き続き宿泊した場合は、それぞれ1回） |

---

### 10.4 運用上の注意点

#### 除外対象

以下の利用者は計算から除外されます：
- 短期利用居宅介護費を算定する者
- 月途中で登録開始・終了した者（日割り計算）

#### 実務的な対応

```
週4回未満が予想される場合の対応：
├─ 通いの回数を増やす
├─ 訪問を追加する（見守り訪問も可）
├─ 短期的な泊まりを提案
└─ ケアプランの見直し
```

## 11. まとめ

### 11.1 定員の3つのポイント

```
1. 登録定員（29人以下）
   └─ 事業所に登録できる利用者の総数
   
2. 通い定員（登録の1/2〜15人）
   └─ 同時に通いサービスを利用できる人数
   └─ 前半・後半それぞれでカウント
   
3. 泊まり定員（通いの1/3〜9人）
   └─ 同時に宿泊サービスを利用できる人数
```

---

### 11.2 重要な解釈

**「同時利用者数」が基準**

- 延べ人数ではない
- 時間帯別に利用者が入れ替わっても問題なし
- プロジェクトBでは前半・後半を別々にカウント

---

### 11.3 弾力運用

- 特に必要と認められる場合は定員超過も可能
- ただし、恒常的な超過は不可
- プロジェクトBでは警告表示（禁止ではない）
