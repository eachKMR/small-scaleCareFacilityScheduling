## 1. 居室管理の本質

### 1.1 基本的な前提

**夜勤者は1人で全員を見る**

小規模多機能型居宅介護では、基本的に以下の人員配置が定められています：

| 時間帯 | 配置基準 |
|--------|---------|
| **夜勤** | 1名以上 |
| **宿直** | 1名以上（訪問体制がない場合） |

---

### 1.2 実態：100%が1人体制

日本医療労働組合連合会の「2024介護施設夜勤実態調査」によると：

```
調査対象：小規模多機能型居宅介護 27事業所
結果：27事業所すべて（100%）が1人体制で夜勤
勤務形態：27事業所すべて2交代制
```

**つまり**：
- 夜間は**必ず1人**で対応
- 最大9人の宿泊者を1人で見守る
- 緊急時も1人で判断・対応

---

### 1.3 なぜ居室管理が重要なのか

#### 理由1：リスクマネジメントが必須

夜勤者1人で対応するため、**予定調整の段階でリスクを織り込む**必要があります。

```
予定調整時の考慮事項：
  ├─ 利用者の要介護度・ADL（日常生活動作）
  ├─ 夜間の見守り頻度（排泄、徘徊リスク等）
  ├─ 医療的ケアの必要性
  ├─ 認知症の状態（BPSD：行動・心理症状）
  ├─ 他の宿泊者との相性
  └─ 緊急時の対応可能性
```

#### 理由2：安全性の担保

人の命を預かる施設運営において、**1人体制での安全性担保**は極めて重要です。

**具体的なリスク**:
- 利用者の急変（体調不良、転倒、誤嚥等）
- 徘徊・転倒リスク
- 複数の利用者から同時にコール
- 災害等の緊急事態

**定員の法的枠組みについては、L0_業務_定員の法的枠組み.md を参照してください。**

## 2. 夜勤1人体制の現実

### 2.1 夜勤の業務内容

```
16:00 出勤、引き継ぎ
17:00 夕食準備、食事介助
19:00 口腔ケア、就寝介助
21:00 消灯
      ↓
  【夜間業務】
  ├─ 1〜2時間おきに安否確認（巡回）
  ├─ コール対応
  ├─ 体位変換
  ├─ 排泄介助
  ├─ 記録作成
  └─ 緊急時対応
      ↓
6:00 起床介助
7:00 朝食準備、食事介助
10:00 引き継ぎ、退勤
```

**休憩**：1〜2時間（ただし、コールは見逃せない）

---

### 2.2 夜勤の困難さ

#### 困難1：緊急時の1人対応

**利用者の急変時**:
```
夜勤者1人の場合：
1. 急変した利用者に対応
2. 同時に他の利用者の見守りが手薄に
3. 救急車の手配・医師への連絡
4. 家族への連絡
5. 記録の作成

→ すべてを1人で判断・実行
```

**複数人体制との違い**:
```
複数人（2人）の場合：
1人目：急変した利用者に専念
2人目：電話連絡、他の利用者の落ち着かせ
→ 役割分担で対応スピード向上、安全性向上
```

---

#### 困難2：心理的負担

**職員の声**（求人サイト等の調査より）:

> 「忙しい時や緊急時に1人で対応しなくてはいけない」
> 
> 「夜勤中は常に1人という緊張や孤独感から、心身の不調を引き起こす場合もある」
> 
> 「利用者さんからのコールを見逃すことはできないため、気が休まらず、全く仮眠が取れない」

---

#### 困難3：休憩が取れない

- 法的には休憩時間があるが、実態は取りにくい
- コールがあればすぐに対応しなければならない
- 複数の利用者が夜間不穏になると、休憩ゼロのこともある

---

### 2.3 「ワンオペ夜勤」のメリット・デメリット

#### メリット

| メリット | 説明 |
|---------|------|
| **マイペースで仕事** | 他の人と合わせる必要がない |
| **落ち着いた時間を活用** | 記録作成、研修資料作成等 |
| **夜勤手当** | 1回3,000〜8,000円 |

#### デメリット

| デメリット | 説明 |
|-----------|------|
| **緊急時の1人対応** | 判断・実行をすべて1人で |
| **心理的負担** | 孤独感、常に緊張状態 |
| **休憩が取れない** | コールを見逃せない |
| **体調を崩しやすい** | 生活リズムの乱れ |

## 3. 予定調整の制約と課題

### 3.1 定員の制約

```
泊まり定員：最大9人

問題：
定員が埋まると、新たな宿泊希望があっても受け入れられない
```

**実際の声**（利用者・家族向けサイトより）:

> 「小規模多機能型居宅介護の中の泊まりサービスの定員がいっぱいになると、泊まりのサービスを受けたくても泊まれない利用者が出てしまうこともあります」

---

### 3.2 突発的な宿泊対応の難しさ

#### ケース1：家族の緊急事態

```
当日の流れ：
9:00 通いサービスで来所
14:00 家族から連絡「体調不良で今夜見られない」
     ↓
  急遽、宿泊に変更
     ↓
  居室の確保、夕食・朝食の手配
  夜勤者への引き継ぎ
  着替え・洗面用具等の準備
```

**課題**:
- 定員に空きがあるか即座に判断
- 夜勤者の負担を考慮（既に何人泊まっているか）
- 他の宿泊者との相性

---

#### ケース2：日中に体調変化

```
15:00 通いサービス利用中に体調変化
     ↓
  「今夜は様子を見たい」
     ↓
  そのまま宿泊へ
```

**柔軟性の利点**:
- 環境変化が少ない（移動不要）
- 同じスタッフが対応（安心感）

**調整の難しさ**:
- 既に定員まで予約が入っている場合
- 夜勤者の負担増

---

### 3.3 職員不足の現実

小規模多機能型居宅介護では、職員不足が深刻な課題となっています。

**影響**:
- 人員配置がギリギリ
- 1人の職員の業務負担が大きい
- 教育体制が不十分
- 夜勤者の心理的負担が大きい

**詳細な職員充足状況については、L0_業務_調整業務の制約.md の「6.1 職員の充足状況」を参照してください。**

---

### 3.4 登録率の低さ（経営課題）

小規模多機能型居宅介護では、登録定員に対する登録率が低い傾向にあります。

**理由の推測**:
- 認知度不足
- ケアマネージャー変更のハードル
- 他サービス併用不可
- **宿泊定員が埋まりやすく、柔軟性が損なわれる**

**詳細な登録率データと新規利用者獲得の戦略については、L0_業務_新規利用者獲得.md の「1.2 理想的な登録率」を参照してください。**

## 4. 居室管理で考慮すべきリスク

### 4.1 利用者の状態によるリスク

| リスク分類 | 具体例 | 夜勤対応の負担 |
|-----------|--------|--------------|
| **徘徊リスク** | 認知症で夜間徘徊 | 頻繁な見守り必要 |
| **転倒リスク** | 歩行不安定、夜間トイレ | 巡回頻度を増やす |
| **医療的ケア** | インスリン注射、吸引等 | 医療的判断が必要 |
| **BPSD** | 夜間不穏、大声、暴力 | 他利用者への影響大 |
| **排泄介助** | 頻回なオムツ交換 | 体力的負担 |

---

### 4.2 複数利用者の組み合わせリスク

```
例：ある日の宿泊者構成

利用者A：認知症、夜間徘徊あり
利用者B：夜間不穏、大声を出す
利用者C：2時間おきの体位変換必要
利用者D〜I：その他6名

→ 夜勤者1人で対応可能か？
```

**調整担当者の判断**:
- A・Bを同じ日にすると、夜勤者の負担が過大
- Cがいる日は、A・Bを避ける
- 比較的安定した利用者を優先的に受け入れる日を作る

---

### 4.3 災害・緊急事態のリスク

**地震等の災害時**:
```
夜勤者1人の場合：
1. 利用者9人の安否確認
2. 避難誘導（車椅子・歩行困難な方含む）
3. 応援職員の到着まで1人で対応

→ 物理的に困難
```

**対策**（予定調整レベル）:
- 自力歩行可能な利用者の比率を考慮
- 車椅子利用者を固めすぎない

## 5. プロジェクトBでサポートすべき業務課題

### 5.1 居室の空き状況の可視化

**現状の課題**:
- 「今日泊まれますか？」の問い合わせに即答できない
- 紙の予定表を見て、数える必要がある

**プロジェクトBでの対応**:
```
日付ごとの居室状況を一目で表示：
  11月25日（月）: 7/9室 使用中 → あと2室空き
  11月26日（火）: 9/9室 使用中 → 満室
  11月27日（水）: 5/9室 使用中 → あと4室空き
```

---

### 5.2 リスクの可視化

**現状の課題**:
- 「この日は夜勤が大変そう」という感覚的な判断
- 記録が残らない

**プロジェクトBでの対応**:
```
利用者ごとにリスク情報を登録：
  ├─ 夜間徘徊リスク: あり/なし
  ├─ 夜間不穏: あり/なし
  ├─ 医療的ケア: あり/なし
  └─ 要介護度: 1〜5

日付ごとに「夜勤負担度」を自動算出：
  11月25日（月）: ★★☆☆☆（やや軽い）
  11月26日（火）: ★★★★★（非常に重い）
  11月27日（水）: ★★★☆☆（標準的）
```

---

### 5.3 突発的な宿泊対応の支援

**現状の課題**:
- 急な宿泊依頼に対して、空きがあるか即座に判断できない
- 夜勤者への負担を考慮する余裕がない

**プロジェクトBでの対応**:
```
「今日泊まれますか？」機能：
1. 今日の居室空き状況を即表示
2. 今日の宿泊者リストを表示
3. 夜勤負担度を表示
→ 調整担当者が総合的に判断
```

---

### 5.4 新規利用者獲得の支援

**現状の課題**:
- 「泊まりが埋まっていて受け入れられない」と断るケースが多い
- 実際には、特定の日だけ埋まっていることもある

**プロジェクトBでの対応**:
```
月間の空き状況を俯瞰：
  ├─ 週末は埋まりやすい
  ├─ 平日は空きが多い
  └─ 特定の日（イベント等）だけ埋まる

→ 「平日なら受け入れ可能」と具体的に提案できる
```

## 6. 居室管理の実務上のポイント

### 6.1 居室の物理的制約

小規模多機能では、宿泊用の個室が確保されています。

**基準**:
- 1人あたり7.43㎡以上
- プライバシーの確保
- 寝具、収納等の設備

**実務上の考慮**:
- 特定の居室を好む利用者
- 隣の居室との相性（音、匂い等）
- トイレへの距離

---

### 6.2 連泊への対応

```
利用者Aの予定：
  11月25日（月）: 泊まり
  11月26日（火）: 泊まり
  11月27日（水）: 泊まり

→ 同じ居室を連続で使用
```

**考慮事項**:
- 居室の清掃タイミング
- 荷物の扱い
- 連泊による精神的安定

---

### 6.3 入退所の記号管理

月別予定表では、宿泊の状況を記号で表します：

| 記号 | 意味 |
|------|------|
| **入** | 入所（宿泊開始） |
| **○** | 宿泊継続中 |
| **退** | 退所（宿泊終了） |

**プロジェクトBでの扱い**:
- 期間で管理（開始日〜終了日）
- 自動的に「入→○→退」を表示

## 7. まとめ

### 7.1 居室管理の重要性（再掲）

```
夜勤者1人で最大9人を見守る
  ↓
予定調整の段階でリスクマネジメントが必須
  ↓
居室管理 = 安全性の担保
```

---

### 7.2 予定調整で考慮すべきこと

| 考慮事項 | 内容 |
|---------|------|
| **定員** | 最大9人、埋まると受け入れ不可 |
| **夜勤負担** | 利用者の状態、組み合わせ |
| **突発対応** | 急な宿泊依頼への柔軟性 |
| **安全性** | 緊急時の対応可能性 |
| **新規獲得** | 空き状況の可視化 |

---

### 7.3 プロジェクトBの役割

プロジェクトBは、**居室管理の実務を支援**します：

1. **空き状況の可視化** - 即座に判断できる
2. **リスクの可視化** - 夜勤負担度を数値化
3. **突発対応の支援** - 今日泊まれるか即答
4. **新規獲得の支援** - 月間の空き状況を俯瞰

**ゴール**:
調整担当者が、安全性を担保しながら、柔軟に宿泊を受け入れられるようにする。
