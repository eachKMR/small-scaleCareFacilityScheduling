## 3. LocalStorage戦略

### 3.1 容量制限

#### 3.1.1 ブラウザ別の制限

| ブラウザ | 容量 |
|---------|------|
| Chrome | 10MB |
| Edge | 10MB |
| Firefox | 10MB |
| Safari (macOS) | 5MB |
| Safari (iOS) | 2.5MB |

**設計上の前提**:
- Safari (iOS)の2.5MBを最小値として設計
- 1ヶ月分のデータ: 約600KB
- 余裕を持って4ヶ月分保持可能

---

#### 3.1.2 データサイズの見積もり

**利用者マスタ（29名）**:
```
1人あたり: 約150バイト
29名: 約4.4KB
```

**月間スケジュール（29名 × 33セル × 2行）**:
```
1セルあたり: 約300バイト
29名 × 33セル × 2行 = 1,914セル
約574KB
```

**備考（100件想定）**:
```
1件あたり: 約200バイト
100件: 約20KB
```

**合計**: 約600KB（1ヶ月分）

---

### 3.2 データ保持戦略

#### 3.2.1 基本方針

**ルール**:
- 編集中の月のデータのみ保持
- 他の月のデータは保持しない
- Excel保存により長期保存可能

**詳細は「要件定義書_データ管理_v1.0.md」を参照**

---

#### 3.2.2 容量超過時の動作

**自動削除**:
```javascript
try {
  localStorage.setItem(key, data);
} catch (e) {
  if (e.name === 'QuotaExceededError') {
    // 最も古い schedule_YYYY-MM を削除
    const oldestKey = findOldestScheduleKey();
    localStorage.removeItem(oldestKey);
    
    // 再試行
    localStorage.setItem(key, data);
  }
}
```

**削除対象**:
- `schedule_YYYY-MM` キーのみ
- `users` と `notes` は削除しない

---

### 3.3 データの暗号化

#### 3.3.1 現状（Phase 0-1）

**暗号化なし**

**理由**:
- LocalStorageはブラウザのサンドボックス内
- 個人情報は氏名のみ（生年月日、住所などなし）
- 機密性は低い

---

#### 3.3.2 将来的な拡張（Phase 2以降）

**検討事項**:
- 暗号化の実装（AES-256）
- パスワード保護
- データのバックアップ

**実装例**:
```javascript
// Web Crypto APIを使用
async function encryptData(data, password) {
  const encoder = new TextEncoder();
  const dataBuffer = encoder.encode(JSON.stringify(data));
  
  const key = await deriveKey(password);
  const iv = crypto.getRandomValues(new Uint8Array(12));
  
  const encrypted = await crypto.subtle.encrypt(
    { name: 'AES-GCM', iv },
    key,
    dataBuffer
  );
  
  return {
    encrypted: arrayBufferToBase64(encrypted),
    iv: arrayBufferToBase64(iv)
  };
}
```

## 4. セキュリティ

### 4.1 データの安全性

#### 4.1.1 LocalStorageの制限

**同一オリジンポリシー**:
- LocalStorageは同一オリジンからのみアクセス可能
- 他のWebサイトからはアクセス不可

**ブラウザのサンドボックス**:
- ブラウザのセキュリティ機構により保護
- 他のアプリケーションからはアクセス困難

---

#### 4.1.2 XSS対策

**ユーザー入力のサニタイズ**:
```javascript
function sanitizeInput(input) {
  const div = document.createElement('div');
  div.textContent = input;
  return div.innerHTML;
}
```

**HTML生成時のエスケープ**:
```javascript
function escapeHtml(text) {
  const map = {
    '&': '&amp;',
    '<': '&lt;',
    '>': '&gt;',
    '"': '&quot;',
    "'": '&#039;'
  };
  return text.replace(/[&<>"']/g, m => map[m]);
}
```

---

#### 4.1.3 CSP（Content Security Policy）

**推奨設定**:
```html
<meta http-equiv="Content-Security-Policy" 
      content="default-src 'self'; 
               script-src 'self' https://cdn.sheetjs.com; 
               style-src 'self' 'unsafe-inline';">
```

**説明**:
- default-src 'self': 同一オリジンのみ許可
- script-src: SheetJSのCDNを許可
- style-src: インラインCSSを許可（'unsafe-inline'）

## 5. パフォーマンス

### 5.1 描画の最適化

#### 5.1.1 仮想スクロール（将来的に検討）

**問題**:
- 58行 × 33列 = 1,914セルの描画
- 全セルをDOMに追加すると重い

**解決策**:
```javascript
// 表示領域のセルのみ描画
class VirtualScroll {
  render() {
    const visibleRows = this.getVisibleRows();
    const fragment = document.createDocumentFragment();
    
    visibleRows.forEach(row => {
      const rowElement = this.createRow(row);
      fragment.appendChild(rowElement);
    });
    
    this.container.innerHTML = '';
    this.container.appendChild(fragment);
  }
}
```

---

#### 5.1.2 DocumentFragmentの使用

**効率的なDOM操作**:

function renderGrid(users, dates) {
  const fragment = document.createDocumentFragment();

    fragment.appendChild(dayStayRow);

  gridContainer.innerHTML = '';

### 5.3 メモリ管理

#### 5.3.1 タイムアウトタイマーのクリア

**問題**:
- 削除⇔復元のタイマーが蓄積するとメモリリーク

**解決策**:
```javascript
class CellEditor {
  constructor() {
    this.restoreTimers = new Map();
  }
  
  deleteCell(cell) {
    // 既存のタイマーをクリア
    const key = `${cell.userId}_${cell.date}`;
    const existingTimer = this.restoreTimers.get(key);
    if (existingTimer) {
      clearTimeout(existingTimer);
    }
    
    // 新しいタイマーを設定
    const timerId = setTimeout(() => {
      cell.clearDeletedState();
      this.restoreTimers.delete(key);
    }, AppConfig.RESTORE_TIMEOUT);
    
    this.restoreTimers.set(key, timerId);
  }
  
  cleanup() {
    // 全タイマーをクリア
    this.restoreTimers.forEach(timerId => clearTimeout(timerId));
    this.restoreTimers.clear();
  }
}
```

---

#### 5.3.2 イベントリスナーの削除

**問題**:
- 画面遷移時にイベントリスナーが残るとメモリリーク

**解決策**:
```javascript
class Component {
  constructor() {
    this.eventListeners = [];
  }
  
  addEventListener(element, event, handler) {
    element.addEventListener(event, handler);
    this.eventListeners.push({ element, event, handler });
  }
  
  destroy() {
    this.eventListeners.forEach(({ element, event, handler }) => {
      element.removeEventListener(event, handler);
    });
    this.eventListeners = [];
  }
}
```
